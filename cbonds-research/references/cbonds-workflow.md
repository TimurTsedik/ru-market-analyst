# cbonds через браузер — точная процедура

cbonds — первичный источник по инструментам. Работа идёт через Claude in Chrome
в браузере пользователя (он уже залогинен под своей подпиской). Поиск ломается на
мелочах — следуй шагам буквально.

## Подключение (один раз за сессию)

1. Подгрузи браузерные инструменты через `tool_search`: запросы вроде
   "browser navigate read page connect Chrome", "get page text", "computer
   screenshot click type", "tabs context". Это загрузит navigate,
   list_connected_browsers, select_browser, tabs_context_mcp, get_page_text,
   find, computer, browser_batch.
2. `list_connected_browsers`. Затем ОБЯЗАТЕЛЬНО спроси пользователя (через
   ask_user_input), какой браузер использовать — перечисли каждый по имени с его
   deviceId плюс опцию «открыть экран подтверждения во всех Chrome». Сам не
   выбирай. По ответу — `select_browser` с нужным deviceId.
3. `tabs_context_mcp{createIfEmpty:true}` → запомни `tabId`.
4. Доступ к домену: расширение спросит разрешение на `cbonds.ru`. Если придёт
   «Permission denied» — попроси пользователя разрешить cbonds.ru в Chrome
   (всплывашка расширения / значок → разрешить) и повтори. Сам логин не вводишь.
5. Приведи окно к известному размеру (≈1440×900) и zoom=100%, если есть подходящий
   tool (resize/window bounds); иначе зафиксируй текущий размер и не доверяй
   координатам — они «плывут» при ресайзе/зуме.

## Поиск бумаги по ISIN

**Локацию элементов делай через селекторы/DOM (`find`, DOM/JS-tool при наличии), а
не по фиксированным пикселям.** Координаты ниже — крайний
фолбэк (вероятно неверны при ресайзе/зуме). Но **сам ввод — реальными нажатиями**
(`computer` → `type`), не `form_input`: автокомплит cbonds не срабатывает на
программной установке значения.

**Перед поиском прокрути страницу вверх** (поле поиска вверху; после свежего
`navigate`/при прокрученной странице клик попадает мимо — частая поломка из живого
прогона).

Батчем (`browser_batch`) на нужном tabId:
1. Найди поле поиска через `find` по placeholder «Быстрый поиск облигаций, акций,
   фондов, индексов, компаний, ISIN» (фолбэк-координаты ~[600,91]); `computer
   left_click` по нему.
2. `computer key cmd+a` → `key Backspace` (очистить поле).
3. `computer type` <ISIN>.
4. `computer wait 2-3` → **`computer screenshot` — подтверди, что выпадашка появилась
   и ISIN в поле** (без этого не кликай).
5. По скриншоту кликни первый результат нужной вкладки (Облигации / Акции / Индексы /
   Компании / ETF & Funds) — по **фактической** координате результата, не по памяти.
6. `computer wait 2` → `computer screenshot`.

Если выпадашки нет / страница не сменилась — **не кликай вслепую**: скроллни вверх,
перечитай скриншот, повтори. ISIN-gate ниже всё равно поймает чужую карточку.

### ISIN-gate (обязательно, до любой цифры)

**Первым делом на открытой карточке прочитай ISIN и сверь байт-в-байт с ISIN из
портфеля.** Совпал — продолжай; не совпал — **отбрось всё и переищи** (ловушка
серий одного эмитента: ТГК-14 с несколькими выпусками). Заголовок и `tabId.url`
тоже подтверждают бумагу, но решает совпадение ISIN.

## Что считывать (чек-лист по бумаге)

Со сводного блока карточки сними:
- **Название, ISIN, тип** — под заголовком курсивом: «Секьюритизация, Senior
  Secured» / «Структурированные продукты, Дисконтные, Для квал. инвесторов» /
  «Бивалютные облигации» / «Переменная ставка» (= флоутер) / «Гарантированные».
- **Рейтинги** — слева: агентства (Эксперт РА, АКРА, НКР, Dagong, и т.д.) и
  значения. 🚩 **Активно ищи `Withdrawn` / «отозван»** — отзыв рейтинга это
  сигнал, особенно если отозвали несколько агентств.
- **Статус** (В обращении), **Объём**, **Размещение**, **Погашение (оферта)** —
  смотри пут/колл-оферты, они меняют дюрацию.
- **Текущий купон** (% и фикс/плавающий; у флоутера будет «Формула расчёта»).
- **Цена** — % от номинала (indicative). 🚩 Глубокий дисконт при низком купоне =
  длинная дюрация (низкий купон + цена заметно ниже номинала → длинная дюрация).
- **Доходность / дюрация** — YTM/YTP/YTC и дюрация (yield-to-worst; см.
  `../../ru-market-shared/references/instruments-rf.md`). 🚩 Оценивай премию через **G-спред к сроковой ОФЗ**, не
  сравнением YTM с ключевой ставкой напрямую.
- **Ликвидность** (балл). 🚩 Цена «—» = рыночной котировки нет, неликвид (типично
  для структурных нот/ИОС — выйти до погашения может не выйти).
- **НКД**, и для секьюритизаций/амортизируемых — **непогашенный номинал** (face
  может быть < 1000; критично для оценки стоимости позиции).
- Для **бивалютных/USD**: номинал в валюте ($100 или $1000 — проверь, это меняет
  оценку в разы; сверь с рублёвой стоимостью из приложения пользователя).

`get_page_text` иногда отдаёт блок финотчётности (МСФО) вместо цены — для
числовых полей надёжнее скриншот сводного блока, чем текст.

## Ветка «Акции» (если бумага — эквити)

Для акции МСФО/дивидендный блок, который `get_page_text` отдаёт (и который для облигаций
выше помечен как «шум вместо цены»), — это **полезная нагрузка**, а не помеха.

- Вкладка результата — **«Акции»** (не «Облигации»); ISIN-gate тот же.
- Со сводного блока/МСФО сними: **дивиденды** (история, объявленные, отсечка),
  **мультипликаторы** (P/E, EV/EBITDA, ND/EBITDA, P/B), **прибыль/EBITDA/долг** при наличии,
  **free-float**, мажоритарий.
- cbonds по эквити — **indicative/агрегатор**: его мультипликаторы НЕ выдавай за факт —
  пересчитай сам (live-цена/капитализация + число акций/классы из раскрытия + МСФО;
  `../../ru-market-shared/references/equity-fundamentals.md`), cbonds = сверка.
- Что брать вживую помимо cbonds (цена/free-float/дивиденды) и полная иерархия источников —
  `../../ru-market-shared/references/equity-fundamentals.md`.

## Надёжность считывания

- **Числа — со скриншота сводного блока по умолчанию**; `get_page_text` — для
  текстовых полей (он иногда отдаёт блок МСФО вместо цены).
- **Бэнды правдоподобности (для обычных облигаций):** цена 1–300%, YTM/купон
  0–100%, дюрация 0–30 лет. Выход за бэнд → **не отбрасывай автоматически**:
  перечитай скриншотом и, если число реальное (структурный/дефолтный/глубоко-
  дисконтный кейс) — пометь как особый случай.
- **Статус по бумаге:** помечай verified / failed / skipped. При сбое — один ретрай,
  затем **продолжай**, не блокируя весь разбор. В конце перечисли бумаги, по которым
  данные cbonds получить не удалось.
- **Degraded / без подписки cbonds:** **нет подписки** (cbonds открывается, но только публичная
  часть) — бери из неё что видно + **MOEX ISS** (цена/YTM/дюрация/ликвидность/дивиденды;
  free-float — карточка/раскрытие; `../../market-data/references/market-data.md`) + **сайты
  агентств** (рейтинги Эксперт РА/АКРА/НКР) + раскрытие эмитента (e-disclosure). **cbonds совсем
  недоступен** — то же, но без cbonds. Везде low-confidence/single-source, sanity-gate строже;
  **не выдумывай по памяти**. Полная иерархия фолбэка — `../../ru-market-shared/references/sources.md`
  («Без подписки cbonds»).

## Глубже по спорным именам

- **Динамика цены** — иконки графика на левом рейле карточки (переключение
  цена/доходность/период).
- **Новости эмитента** — со страницы эмитента / раздел новостей по бумаге.
- **Новости рынка** — верхнее меню «НОВОСТИ & RESEARCH HUB».
- **Проспект / эмиссионные документы** — раздел «Эмиссионные документы» (Условия
  эмиссии / Решение о выпуске / Программа). Поднимай для payoff структурки, условий
  оферты, амортизации, ковенантов. **Скачивание:** клик по ссылке может НЕ запустить
  загрузку → надёжнее **`navigate` прямо на URL загрузки**. Сначала **возьми
  фактический href со страницы** (через `find`/ссылку), не угадывай id; URL вида
  `cbonds.ru/emissions/download/<id>/<bond>/` (сессия залогинена) → zip в Downloads →
  `unzip` → `pypdf` для текста. Если прямой `navigate` не сработал — сохрани скриншот/
  текст ссылки и фолбэк на ручную/браузерную загрузку. **Учти:** для части структурок конкретный payoff в этих
  документах НЕ полон (только каркас) — тогда запроси применимые условия у пользователя
  (см. `../../ru-market-shared/references/instruments-rf.md`, «ИОС»).

## Кэш-стратегия по round-trip

Один поток поиска за `browser_batch` (поиск → выбор → скриншот). Не пытайся открыть
несколько бумаг одним батчем — выпадашка по каждой своя. Локацию — через селекторы;
координаты ~[555–560, 184] только как фолбэк: если съехало — скриншоть и кликай по факту.
