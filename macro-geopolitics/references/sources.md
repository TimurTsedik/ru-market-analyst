# Реестр источников для макро-/гео-анализа (гибрид: дефолты → вето пользователя)

В геополитике **выбор источника и есть половина анализа** — ангажированность зашкаливает с
любой стороны. Поэтому здесь не «список правды», а **сбалансированный по линзам реестр** с
явными метками: тип, линза, характерный уклон. Помечено `[veto?]` = кандидат, который
пользователь с наибольшей вероятностью захочет исключить или понизить в весе — **вычеркни/
добавь под себя** (это гибрид-модель: я набросал, ты редактируешь).

## Принципы работы с источниками (важнее самого списка)

1. **Читай поперёк линз, не из одной.** Любой одиночный нарратив = single-source →
   low-confidence (тот же sanity-gate, что в `../../ru-market-shared/references/sources.md`).
2. **Отделяй факт от интерпретации.** «Что произошло» (поддаётся кросс-проверке) ≠ «что это
   значит» (интерпретация, ангажирована). Для фактов ищи ≥2 независимые линзы.
3. **Взвешивай по калибровке, не по громкости.** Доверие источнику ∝ его проверяемый
   трек-рекорд, а не яркость/совпадение с твоим страхом.
4. **Рынок — арбитр поверх нарративов.** Где есть рыночно-вменённый сигнал
   (`indicators-signals.md`), он бьёт публицистику: деньги калиброваннее колонок.
5. **Дата и провенанс.** Каждый material-факт — источник + as-of; режимные акты
   (капконтроль/санкции) быстро устаревают — перечитывай первоисточник.
6. **Свежесть = сегодня.** При наличии дневного ряда (курс/RGBI/недельная инфляция) бери
   значение **на текущую дату**, а не первый снипет из выдачи; не путай дату публикации с
   датой данных. Полный протокол — `critical-search.md`.
7. **Не выдумывай при блоке.** Источник отдал 403/paywall/пустой JS → альтернатива или
   пометка «не верифицировано вживую» + спроси пользователя. Никаких чисел по памяти.

## A. Твёрдые данные (низкая интерпретируемость — основа)

| Источник | Что | Метка |
|---|---|---|
| ЦБ РФ (cbr.ru) | ставка, RUONIA, курс, инфляц. ожидания, нормативы FX | офиц. РФ; первичен по ДКП/курсу |
| Росстат | ИПЦ, ВВП, пром. производство | офиц. РФ; **ценовые контроли могут искажать ИПЦ** — кросс-чек дефицитами |
| Минфин РФ | бюджет, дефицит, ФНБ, план размещений | офиц. РФ |
| MOEX ISS | кривая ОФЗ, RGBI, цены/объёмы, инструменты недвиж. | рыночные данные, первичен по цене |
| Росреестр | объёмы сделок с недвижимостью | офиц. РФ; ключевой для ликвидности жилья |
| cbonds | спреды, рейтинги, термины выпусков | агрегатор (см. shared/sources.md) |

> Официальная статистика РФ — первична по «твёрдым» числам, но при ценовых контролях/
> военной экономике может занижать инфляцию и приукрашивать; **держи теневые индикаторы**
> (дефициты, нормирование, объёмы сделок) как перекрёстную проверку.

## B. Геополитические линзы (высокая интерпретируемость — читать поперёк)

Цель — покрыть спектр, а не назначить «правильную» линзу. Минимум: по одной из каждой
группы, сверять между собой.

| Линза | Характер | Метка |
|---|---|---|
| Нейтральные ленты (Reuters, AP, AFP, Bloomberg) | оперативные факты, рыночный фокус | прозападный мейнстрим-уклон, но высокая фактологич. калибровка |
| Западные аналитические (FT, Economist, WSJ, think-tanks: ISW, RUSI, IISS) | интерпретация/прогноз | проукраинский/западный уклон `[veto?]` — полезны как фактура, не как вердикт |
| Госаффилированные РФ (ТАСС, РИА, РБК) | офиц. позиция РФ, эконом. детали | прогос-уклон `[veto?]` — ценны по внутр. экономике/решениям властей |
| Независимые/эмигрантские РФ (по выбору пользователя) | альтернативная внутр. оптика | антигос-уклон `[veto?]` — ценны по теневым издержкам/настроениям |
| Финансовая аналитика по РФ (брокеры РФ, телеграм-аналитики рынка) | рынок/ставка/FX | продуктовый/позиционный уклон `[veto?]` — проверяй на конфликт интереса |
| OSINT/конфликт-трекеры (мониторы линии фронта, верификаторы) | геопривязка событий | разнятся по методологии `[veto?]` — для S4-локаций; помечай confidence |

## C. Рыночно-вменённые (арбитр поверх B)
См. `indicators-signals.md` §«Рыночно-вменённые сигналы»: USD/RUB·CNY/RUB, кривая ОФЗ,
RGBI, G-спреды, суверенные прокси, объёмы/индексы жилья, ставки вкладов. Это «вероятность по
мнению денег» — кросс-чек любой публицистике.

## Как пользователю редактировать этот реестр (вето-протокол)
- Вычеркни линзы, которым не доверяешь, — но **оставь покрытие спектра**: если убрать все
  «неудобные» линзы, анализ схлопнется в эхо-камеру и потеряет ценность.
- Добавь свои доверенные источники с **явной меткой уклона** (даже доверенный источник
  имеет уклон — назови его).
- Помечай источники, которым доверяешь по *фактам*, но не по *интерпретации* (частый случай).
- Для запертого рынка/санкций понижай вес внешних прокси (могут быть оторваны) — см.
  оговорку в `indicators-signals.md`.

## Блок вывода «Источники и аудит» (как в наборе)
В конце разбора — компактный блок по ключевым фактам/сигналам:
```
Источники и аудит
- Режим (ставка/инфляция/FX): <источник> — as-of <дата>
- Геофакт <X>: подтверждён линзами <…> (≥2 независимые) | single-source
- Рыночно-вменённый сигнал <Y>: <значение/направление> — as-of <дата>
- Вероятности сценариев: суждение аналитика на основании <…>, не факт
```

*Источник в геополитике — половина анализа. Читай поперёк линз, отделяй факт от
интерпретации, взвешивай по калибровке, давай рынку быть арбитром. Реестр — под вето
пользователя; сохраняй покрытие спектра.*
