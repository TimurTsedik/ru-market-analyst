# Карта «сценарий → актив», робастное решение, шок-вектор для MC, триггеры

Этот файл превращает сценарное дерево (`geopolitical-scenarios.md`) и индикаторы
(`indicators-signals.md`) в **решение** и в **стресс-входы для Monte-Carlo** (`risk-strategy`).

## Шаг 0 — определи доступное множество назначений (первый фильтр)

Любая карта активов считается **внутри того, что физически доступно** пользователю. Сначала
зафиксируй множество (это меняет всё):

- Вывод капитала за рубеж: **открыт / закрыт / частично** (капконтроль)?
- Если закрыт — реальные назначения сжаты до: **рынок РФ** (ОФЗ/корп/эквити/фонды),
  **недвижимость**, **золото** (физ./обезличенное), **кэш/вклады** (руб/валюта внутри РФ),
  **потребление**. Сравнивать «фантики» нужно внутри *этого* набора, а не с зарубежным
  портфелем, которого нет.
- Потребность в ликвидности и горизонт — как ограничения сверху.

Без этого шага вопрос «продавать/держать» нерешаем — не с чем сравнивать.

## Шаг 1 — матрица «класс актива × сценарий»

Заполняй знаками воздействия (↑↑/↑/→/↓/↓↓) под текущий режим; качественно, с основанием.
Каркас:

| Класс актива (из доступного множества) | S1 стейлмейт | S2 эскалация | S3 деэскалация | S4 хвост-вниз локальн. | S5 шок капконтроля |
|---|---|---|---|---|---|
| ОФЗ-ПК / линкеры (короткий риск ставки) | → / ↑ | → | ↓ (отстаёт) | → | → |
| Длинный фикс ОФЗ/корп | → | ↓↓ (рост ставок/риска) | ↑↑ (re-rate) | → | ↓ |
| Эквити РФ (экспортёры/pricing-power) | → | ↓ (но FX-хедж у экспортёров) | ↑↑ | ↓ | → |
| Золото | ↑ | ↑↑ | → / ↓ | ↑ | ↑ (внутр. хедж) |
| Недвижимость (конкретная локация) | → ном./→ реал. | ↓↓ ликвидн.+реал. цена | ↑ re-rate | **↓↓↓ → ~0** (физ./юрисдикц.) | → (заперта, но реальна) |
| Кэш/вклады руб (высокая ставка) | ↑ (реальн. ставка) | ↓ (девальвация) | ↓ (упуск. re-rate) | → | → |
| Валюта внутри РФ (если доступна) | → | ↑ | ↓ | → | ↓ (конвертация под угрозой) |

Ключевые уроки каркаса (проверяй на живых данных, не как догму):
- **Недвижимость** — реальный актив, но несёт **неликвидность + концентрацию +
  физический/юрисдикционный хвост (S4)**. Её слабость максимальна ровно в той ветке, где
  страх острее. Одна локация = единая точка отказа.
- **Ликвидность дешевеет в эскалации** (S2): уходить из неликвида надо *рано*, если веришь
  в эту ветку — после заморозки рынок не даёт уйти у справедливой цены.
- **Капконтроль (S5)** бьёт по «носимым» хеджам (валюта/вывод), но не по реальным активам
  внутри страны — сдвигает предпочтение к золоту/недвижимости/локальному рынку.

## Шаг 2 — робастное решение (не ставка на один исход)

Поскольку исход неизвестен, выбирай решение, **защищённое по всем колонкам**, а не
оптимальное под одну. Практическая логика «реальный актив ↔ store-of-value»:

- Есть **реальное назначение выручки** (доступный лучший по режиму актив) И склонность к
  «хуже/дольше» (S2/S4↑) → **разгружай неликвид/концентрацию рано**, пока рынок функционирует.
- Назначения **нет** (только дешевеющий кэш под капконтролем) → реальный актив (в т.ч.
  недвижимость) может быть лучшим «фантиком»; **не продавай в панике**, но держи актив
  «готовым к сделке» и переключайся по триггеру.
- **Худший квадрант, избегать:** продать в замерзающий рынок с фаer-sale-дисконтом И
  запарковать в активе, который не лучше (дешевеющий кэш) — потеря стоимости дважды.
- **Хвост S4 (локация → ~0):** это аргумент за **диверсификацию прочь от единственной
  точки отказа**, а не за «всё в ноль». Низкая p × катастрофич. воздействие → частичное
  действие/хедж/триггер, даже если p мала.
- **Частичность.** Не обязательно «всё или ничего»: продать/переложить *часть* концентрации
  часто доминирует по робастности над полным выбором одной ветки.

## Шаг 3 — шок-вектор для Monte-Carlo (handoff в risk-strategy)

Каждый сценарий переводится в **вектор шоков** — стресс-входы для квант-тира
(`../risk-strategy/references/quant-scenarios.md`, тулкит `../risk-strategy/quant/`). Схема:

```
scenario_shock = {
  curve_level_bp:   <сдвиг уровня кривой ОФЗ, бп>        # S2: +, S3: −
  curve_slope_bp:   <изменение наклона 2/10, бп>
  fx_pct:           <шок USD-RUB/CNY-RUB, %>             # S2: +ослабление
  credit_spread_bp: <расширение G-спреда, бп>            # S2/S4: +
  equity_drawdown_pct: <просадка эквити РФ, %>
  realestate_haircut_pct: <хейркат недвижимости, %>      # S4 локальный: до ~100 по затронутой локации
  liquidity_factor: <множитель дисконта при срочной продаже>  # эскалация → выше
  prob_range:       <p-диапазон сценария с основанием>
}
```

Правила handoff:
- Шоки задаются **относительно живой базы** (текущая кривая/курс/спреды из `../market-data/`),
  не абсолютами из памяти.
- MC прогоняется по сценариям → **ES/CVaR · перцентили · P(loss) · fan-chart** под каждый;
  взвешивай исходы p-диапазонами как **суждение**, не как точную меру.
- MC — **вход** в решение, не замена; каждый прогон с **флагами модельного риска**
  (особенно для S4/S5, где распределения толстохвостые и плохо оценены).
- Недвижимость в MC — отдельной ногой с явным `realestate_haircut` и `liquidity_factor`;
  не смешивать с ликвидными бумагами как будто продаётся мгновенно у справедливой цены.

## Шаг 4 — триггеры (предкоммит)
Из `indicators-signals.md`: для выбранного решения выпиши 2–5 триггеров
«индикатор → порог(относительно базы) → действие». Это и есть операционализация решения:
ты не угадываешь разворот, ты заранее решил, *что наблюдаемое* его включит.

## Выход
Множество назначений → матрица «актив × сценарий» → **робастное решение** (с явной логикой
реальный-актив↔store-of-value и частичностью) → **шок-вектор** для MC + handoff в
`risk-strategy` → **триггеры**. Всё с основанием; вероятности — суждение; дисклеймер.

*Карта — не приказ. Считай внутри доступного множества назначений; робастность важнее
ставки на исход; хвост → диверсификация, не паника. Шоки — относительно живой базы, MC со
флагами модельного риска.*
