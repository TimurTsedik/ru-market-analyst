# macro-geopolitics — режим, сценарии, хвостовой риск → стресс-входы и триггеры

Этот скилл превращает расплывчатый макро-/геополитический страх («а если всё рухнет»,
«а если регион захватят») в **дисциплину**: сценарное дерево с обоснованными вероятностями
→ наблюдаемые опережающие индикаторы → рыночно-вменённый кросс-чек → карта «сценарий →
актив» → предкоммитнутые триггеры. Это **движок структурированного рассуждения на живых
уликах, а НЕ предсказатель войны.** Его задача — сделать пользователя хорошо
позиционированным по всем сценариям и хорошо отмониторенным, а не «угадать исход».

## Рамки и безопасность (не обсуждаются)
- **Это разбор/мысль, не ИИР и не геополитический прогноз.** Дисклеймер — обязательный
  пункт вывода. После каждого тезиса — **основание** (данные/источник+as-of/допущение).
  Размер, цена, тайминг, исполнение — за пользователем.
- **Прогноз событий — домен низкой предсказуемости.** Не присваивай событийным исходам
  ложно-точные вероятности; давай **диапазоны с основанием** и помечай как суждение, не факт.
  Где возможно — сверяй с **рыночно-вменённой** оценкой (рынок уже заложил какой-то хвост).
- **Никакого хардкода под «сегодня».** Ставку/курс/инфляцию/спреды/режим/новостной фон
  читай **вживую** (ЦБ/Росстат/MOEX/web_fetch), не из памяти. Скилл работает в любом
  состоянии мира — он задаёт *как думать*, а не *что сейчас происходит*.
- **Anti-anchoring — ядро модуля.** Пользователь часто приходит с ярким сценарием
  («регион захватят → 0»). Оставайся на данных: дай **факт + контраргумент + базовую
  частоту**, не подыгрывай яркой картинке и не спорь ради спора. Один вывод — не сценарий;
  всегда дерево.
- **Приватность:** не выноси личные активы/локации во внешние сервисы сверх необходимого
  для анализа.

## Содержимое (references)
- `references/macro-regime-doctrine.md` — как **вживую** охарактеризовать текущий
  макрорежим РФ (инфляция, ставка, FX-режим, интенсивность капконтроля, фискальный
  импульс, признаки военной экономики) → импликации по классам активов.
- `references/geopolitical-scenarios.md` — **методология сценарного дерева**: базовые
  частоты, anti-anchoring, вероятность-с-основанием, рыночно-вменённый кросс-чек,
  и **сид-библиотека сценариев** (эскалация / стейлмейт / деэскалация + хвостовые ветки).
- `references/indicators-signals.md` — **наблюдаемые опережающие индикаторы** (live, без
  хардкода) и **рыночно-вменённые сигналы** (USD/RUB-форвард, кривая ОФЗ, суверенные
  спреды-прокси, объёмы сделок/индексы по жилью) с рецептами чтения.
- `references/scenario-asset-map.md` — карта **«сценарий → класс актива»** под
  **ограниченное множество назначений** (рынок РФ / недвижимость / золото / кэш /
  потребление, когда вывод капитала закрыт), **схема шок-вектора** для MC и **триггеры**.
- `references/sources.md` — **реестр источников** (гибрид: дефолты сбалансированы по
  линзам, помечены ангажированность и `[veto?]`-кандидаты на исключение пользователем).

## Интейк (до сценариев)
Спроси/уточни: **множество реальных назначений капитала** (что физически доступно: рынок РФ,
недвижимость, золото, кэш, потребление — вывод за рубеж открыт/закрыт?), **горизонт**,
**потребность в ликвидности**, **конкретные хвостовые риски, которые волнуют** (чтобы
сценарная библиотека покрывала их, а не абстракции). Без множества назначений вопрос
«продавать/держать» нерешаем — сравнивать не с чем.

## Воркфлоу (фазы)
1. **Режим (live)** → следуй `references/macro-regime-doctrine.md`: охарактеризуй текущий
   режим по живым данным (ставка ЦБ/инфляция/FX/капконтроль/фиск) — каждое с as-of.
   Рыночные числа бери через `../market-data/SKILL.md`.
2. **Сценарное дерево** → следуй `references/geopolitical-scenarios.md`: построй 3–5
   взаимоисключающих сценариев, присвой **вероятности-с-основанием** (диапазоны), приложи
   **базовую частоту** и **контраргумент** к каждому; явно отметь сценарии пользователя
   и проверь их на anchoring.
3. **Индикаторы + рыночный кросс-чек** → следуй `references/indicators-signals.md`: для
   каждого сценария — **опережающие индикаторы**, что сдвинут его вероятность; сверь с
   **рыночно-вменённым** сигналом (что уже в цене FX-форварда/кривой/спредов/объёмов жилья).
   Расхождение «нутро vs рынок» — само по себе вывод.
4. **Карта → актив + триггеры** → следуй `references/scenario-asset-map.md`: отобрази
   каждый сценарий на классы активов из **доступного множества назначений**; выдай
   **робастное решение** (защищённое во всех колонках) и **предкоммитнутые триггеры**
   (если индикатор X пробил Y → действие Z).
5. **(Full) Handoff в квант** → передай сценарии как **стресс-входы** в
   `../risk-strategy/references/quant-scenarios.md` (Monte-Carlo): шок-вектор
   (уровень/наклон кривой, FX, кредит-спред, просадка эквити, хейркат недвижимости) из
   `scenario-asset-map.md` → ES/CVaR/перцентили/P(loss)/fan-chart под сценарии. MC — **вход**
   в решение, не замена суждению; каждый прогон с флагами модельного риска.

## Выход (handoff)
Режим (с as-of) → **сценарное дерево** (вероятности-с-основанием + контраргумент + базовая
частота) → таблица **опережающих индикаторов** + **рыночно-вменённый кросс-чек** → карта
**«сценарий → актив»** под доступное множество назначений → **робастное решение** +
**триггеры** → (опц.) **шок-вектор для MC** и handoff в `risk-strategy` → блок «Источники
и аудит» → дисклеймер. Material-числа — с источником+as-of; вероятности — как суждение
с основанием, не как факт.

## Когда НЕ использовать
Только котировка/ставка/курс/кривая → `market-data`. Разбор одной бумаги по ISIN →
`cbonds-research`. Парсинг отчёта → `broker-parse`. Корзины/ТОП-5 по готовому снимку →
`risk-strategy` (этот модуль кормит его сценариями, но не заменяет). Чисто личный
не-инвестиционный вопрос без актива на кону — вне набора.

## Definition of Done
- [ ] Режим охарактеризован **вживую** (ставка/инфляция/FX/капконтроль/фиск), каждое с as-of.
- [ ] Построено 3–5 **взаимоисключающих** сценариев; каждому — вероятность-диапазон **с основанием**, базовая частота и **контраргумент**.
- [ ] Сценарии пользователя явно помечены и проверены на **anchoring** (не приняты на веру).
- [ ] К каждому сценарию — **опережающие индикаторы**; сделан **рыночно-вменённый кросс-чек**, расхождение «нутро vs рынок» названо.
- [ ] Решение **робастно** (защищено по всем сценариям), а не ставка на один исход; даны **триггеры** (индикатор→порог→действие).
- [ ] Сравнение идёт внутри **доступного множества назначений** (не предполагается свободная мобильность капитала, если её нет).
- [ ] (Full) При высокой цене решения — сценарии переданы в `risk-strategy` как **шок-вектор** для MC; результат подан со флагами модельного риска.
- [ ] Источники сбалансированы по линзам; блок «Источники и аудит» на месте; вероятности поданы как суждение, не факт.
- [ ] После рассуждений — **основание**; дисклеймер на месте.
