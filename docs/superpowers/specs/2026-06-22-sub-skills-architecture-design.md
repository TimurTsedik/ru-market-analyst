# RU Market Analyst — разбиение на суб-скиллы (design / spec) — ред. 2

Дата: 2026-06-22
Статус: ред. 2 после ревью codex (thread 019eeaf8), на подтверждении

## Цель
5 инвокабельных суб-скиллов + оркестратор + общий reference-пак. Каждый вызывается
**точечно**; полный цикл — через оркестратор **inline-routing** (не через Skill-tool).
Контент переезжает, не переписывается; правила round-1/2 (no-hardcode, амортизация,
ИОС-fallback, рекомендательная таблица, цель+ТОП-5) сохраняются.

## Раскладка репозитория (после)
```
<repo>/
  ru-market-analyst/        # ОРКЕСТРАТОР (тонкий)
  broker-parse/             # references/broker-reports.md
  cbonds-research/          # references/cbonds-workflow.md
  market-data/              # references/market-data.md
  risk-strategy/            # references/{analysis-doctrine, strategy-engine, strategies-bonds, strategies-equity}.md
  portfolio-kb/             # references/portfolio-kb.md (+ monitoring-state)  ← реализуем ПОСЛЕДНИМ, опц.
  ru-market-shared/         # references/{instruments-rf, sources, taxes-accounts}.md
  docs/  README.md
```

## Оркестрация = inline-routing (НЕ Skill-tool)
Оркестратор НЕ «вызывает» суб-скиллы как API. Он **инлайн-маршрутизирует**: на каждой
фазе «**прочитай и следуй инструкциям `../broker-parse/SKILL.md` (+ его references)**»,
далее cbonds-research, market-data, risk-strategy. Оркестратор задаёт **порядок фаз,
требуемые вход/выход (handoff), и финальный DoD** — не полагается на догадки суб-скиллов.

## Общие ссылки и упаковка
- Sibling-относительный путь: `../ru-market-shared/references/<file>.md`. Резолвится при
  per-folder симлинке и при копировании всех соседей.
- **Каждый суб-скилл содержит shared-path-конвенцию:** «общие справочники в
  `../ru-market-shared/references/…`; если их нет — остановись и попроси установить ВСЕ
  ru-market-скиллы».
- **Упаковка/установка:** ставить ВСЕ папки соседями в `~/.claude/skills/`. **Claude.ai:**
  по умолчанию — **только оркестратор** (самодостаточен через свои ссылки на соседей в
  Claude Code; для Claude.ai shared/суб-скиллы можно вендорить в один бандл, если платформа
  принимает мульти-папочный бандл — проверить). **Отдельные zip суб-скиллов НЕ
  поддерживаем** (sibling-ссылки не резолвятся). README это фиксирует честно.

## Черновики frontmatter-описаний — ТОЛЬКО ТРИГГЕР (симптомы, без процесса)
(Процесс/handoff живут в теле SKILL.md, не в описании — иначе модель «срежет» по описанию.)

- **ru-market-analyst:** «Используй для сквозного разбора портфеля РФ: когда присланы
  брокерские отчёты (ТБанк/Альфа/Сбер) или просят свести портфель по брокерам, оценить
  риски целиком, решить что поменять, построить стратегию по всему портфелю. Для
  точечных задач (одна бумага, только цена, только парсинг) — суб-скиллы ru-market-*.»
- **broker-parse:** «Используй, когда нужно распарсить/свести брокерский отчёт РФ (ТБанк
  PDF, Альфа/Сбер XLS, скрин приложения) в чистые позиции. Не для оценки/стратегии/cbonds.»
- **cbonds-research:** «Используй, когда спрашивают про конкретную облигацию/бумагу по
  ISIN — её карточку, рейтинг, оферту, проспект на cbonds. Не для полного портфеля и не
  для парсинга отчётов.»
- **market-data:** «Используй, когда нужна только котировка фонда/акции, ключевая ставка,
  курс валюты или кривая ОФЗ (MOEX/ЦБ). Не для разбора инструмента и не для стратегии.»
- **risk-strategy:** «Используй, когда есть готовый снимок позиций РФ и нужны корзины
  риска, режим и стратегия (что-если, варианты замен). Не для парсинга сырых отчётов.»
- **portfolio-kb:** «Используй, когда просят сохранить/обновить базу знаний о портфеле
  или показать, что изменилось с прошлого раза. Опционально.»
- **ru-market-shared:** «Внутренний справочный пак ru-market. Использовать ТОЛЬКО когда
  другой ru-market-скилл просит загрузить общие справочники; не для прямых запросов
  пользователя.» (без слов облигации/налоги/инструменты — чтобы не триггерить.)

Каждый суб-скилл (и оркестратор) содержит в ТЕЛЕ раздел **«Когда НЕ использовать»** —
главная защита от overlap.

## Handoff-схемы (минимальный выход каждого)
- **broker-parse →** позиции (брокер/счёт/ISIN/кол-во/оценка-если-есть) + кэш + плечо +
  сверка с итогом + mixed-dates.
- **cbonds-research →** по ISIN: тип/класс, цена%+НКД, рейтинг(низший)+флаги, оферта/
  амортизация/непогашенный номинал, YTW/дюрация, ликвидность, источник+as-of.
- **market-data →** live-марки фондов/эквити, кривая (2/5/10 ОФЗ), ставка, курс + as-of.
- **risk-strategy →** корзины+агрегаты, метрики до/после, план замен, ТОП-5.
- **portfolio-kb →** канонический снимок состояния + дифф (опц.).

## Тест-матрица discovery (dry-run после сборки)
Проверить, какой скилл триггерится:
- «разбери RU000A10BPF3» → **cbonds-research**
- «дай курс USD и кривую ОФЗ» → **market-data**
- «распарси Альфа XLS» → **broker-parse**
- «что поменять в портфеле» / «свод по брокерам + отчёты» → **ru-market-analyst**
- «корзины и стратегия по вот этим позициям» → **risk-strategy**
- общий запрос НЕ должен будить ru-market-shared.

## Миграция (compat-фаза — не сломать живой скилл)
1. **Создать новые папки** (broker-parse/… , ru-market-shared/…) через `git mv` контента.
   Чтобы не сломать живую установку (симлинк `~/.claude/skills/ru-market-analyst → repo
   root`): **оставить в корне валидный `SKILL.md`-заглушку** (минимальный, указывает на
   `ru-market-analyst/`) до момента, пока новые соседние симлинки не установлены и не
   проверены. Всё на ветке → реверт восстанавливает монолит.
2. Создать SKILL.md каждого суб-скилла (frontmatter-описание + тонкое тело: свои
   references + shared-path-конвенция + «Когда НЕ использовать» + handoff-выход).
3. Переписать оркестратор в тонкий (inline-routing по фазам, контракт выхода, DoD).
4. Поправить ВСЕ кросс-ссылки; прогнать link-check:
   `rg -n 'references/|\.\./ru-market' */SKILL.md */references/*.md`.
5. **Установка соседями:** симлинки `~/.claude/skills/<each>` → папки репо (вкл.
   ru-market-shared). Поставить и **проверить discovery (тест-матрица)**.
6. **Атомарно переключить:** только после успешной проверки — снять старый симлинк
   монолита (если путь оркестратора сменился) / убедиться, что ru-market-analyst указывает
   на новую папку оркестратора.
7. README (корень): набор скиллов + установка (все соседи; Claude.ai — бандл).

## Очерёдность реализации (по codex)
ru-market-shared (+path-конвенция) → broker-parse → cbonds-research → market-data →
risk-strategy → **portfolio-kb (последним, опц.; при сложности — слить в оркестратор/
risk-strategy)** → оркестратор → кросс-ссылки/README/установка/discovery-тест.

## Риски / на что смотреть
- Path-резолв общих ссылок (копия vs симлинк vs Claude.ai-бандл).
- Overlap триггеров (оркестратор vs risk-strategy vs cbonds-research) — лечится узкими
  описаниями + «Когда НЕ использовать» + тест-матрицей.
- Не потерять правила round-1/2 при переезде.
- Живой симлинк-скилл не ломать до атомарного переключения.
- portfolio-kb — возможно избыточен; держать опциональным.

## Definition of Done
- 6 папок-скиллов (5 суб + оркестратор) + ru-market-shared; валидный frontmatter у каждого.
- Точечная инвокация работает (тест-матрица проходит); оркестратор гоняет полный цикл
  inline-routing’ом; нет ложного срабатывания ru-market-shared.
- Общее знание не дублируется; sibling-ссылки резолвятся; link-check чист.
- Все правила прежних раундов сохранены; no-hardcode цел.
- README/установка обновлены; локальные симлинки переключены атомарно.
- Каждый шаг прошёл codex-loop.
