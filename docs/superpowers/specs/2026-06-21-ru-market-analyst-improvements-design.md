# RU Market Analyst — программа улучшений (design / spec) — ред. 2

Дата: 2026-06-21
Статус: ред. 2 после ревью codex (thread 019eeaf8), на подтверждении

## Цель

Устранить все находки многолинзового разбора скилла `ru-market-analyst` (5 линз,
49 находок, 12 тем). Скилл — набор markdown-инструкций (SKILL.md + references),
поэтому «реализация» = редактирование/добавление markdown-файлов. Качество =
доменная корректность (RU fixed income) + надёжность пайплайна.

**Главный риск (по ревью codex):** не превратить прагматичный cbonds-скилл в
тяжёлый data-governance-воркфлоу. Держим в центре: **рыночная оценка брокера +
cbonds**; MOEX/агентства/первоисточники — точечное подтверждение; персистентность
— опциональна.

Принцип каждой задачи: спека → codex-ревью → критическая оценка правок → правка →
подтверждение у codex → в работу → codex ревьюит диф.

### S2 — Источники и аудит (рано: нужно всем шагам)
Файл: `references/sources.md` (новый) + блок в выводе.
- Иерархия **по типу данных** (ред. по codex):
  - стоимость портфеля → **рыночная оценка брокера** (primary mark);
  - исполнимая цена/оборот/ликвидность → **MOEX ISS** (iss.moex.com), когда доступно;
  - термины, рейтинги, амортизация, оферты, ковенанты, новости, проспект → **cbonds** (primary);
  - ставка/макро → cbr.ru; размещения → minfin.gov.ru; рейтинги (подтверждение
    действия) → сайты агентств (raexpert.ru, acra-ratings.ru, ratings.ru, НКР);
    дефолты/события → e-disclosure.ru.
  - Единственный источник = зацепка, не факт (для несущих чисел).
- На каждой **material/action-driving** цифре (та, что двигает вывод/сделку):
  source + as-of (дата котировки И дата чтения). Реализовать **компактным**
  аудит-блоком + правилом материальности — не леджер на каждое число (риск
  утопить вывод).
- Свежесть: `сегодня − дата котировки`; «⚠️ устарела на N дней / последняя
  сделка ДД.ММ» при превышении порога; для неликвидов показывать дату посл. сделки.
- **Mixed-dates** (gap по codex): если отчёты брокеров на разные даты — помечать
  итог «на смешанные даты ДД.ММ–ДД.ММ».
- Sanity-gate: перед негативным ярлыком (слабый кредит, неликвид) — два
  подтверждения ИЛИ пометка «единственный источник, не подтверждено».
- Блок вывода «Источники и аудит»: инструмент → источник(и) → as-of → уверенность
  (confirmed/single-source/estimated); периоды отчётов брокеров; ставка ЦБ as-of.
- web_fetch первоисточников вместо сниппетов для ставки/RGBI/Минфина.

### S3 — Классы инструментов РФ (термины до математики)
Файл: `references/instruments-rf.md` (новый), раздел «Классы».
- Для каждого класса: что это, декомпозиция риска, как котируется на cbonds, в
  какую корзину мапится.
- **Замещайки** (ред.): обычно росправо, номинал/купон привязаны к валюте, выплата
  в рублях по офиц. курсу — **но условия разнятся по проспекту**; добавить
  санкционный/платёжно-цепочечный/юр-док риск; для налога — continuity лота
  (проверить по ст. 219.1, вкл. поправки 2024–2026). Не «просто FX + кредит».
- **ОФЗ-ИН** (ред.): real-rate/инфляционно-индексируемые, **не** общий хедж —
  индексация номинала лагает, остаётся real-yield дюрация, важны breakeven/ликвидность.
- **ОФЗ-ПК** (RUONIA + лаг наблюдения/выплаты) vs **ОФЗ-ПД** (фикс).
- **Суборды/перпы** банков: флаг, **переопределяющий** «A− = ядро»
  (loss-absorption, пропуск/отмена купона, продление колла, списание).
- **ЦФА**; **заблокированные/неторгуемые** (бакет с дисконтом + пометка неликвида).
- **Секьюритизация** (gap по codex): различать эмитента/гаранта/оригинатора/
  сервисера/пул залога; AAA(ru.sf) — **структурное** качество, требовать структурных
  проверок, не приравнивать к корп. senior unsecured.
- **ИОС/структурные**: метод разбора payoff (базовый актив; тип — автоколл/барьер/
  защита капитала; условие купона + даты наблюдения; worst-case тело; кредит
  ЭМИТЕНТА — защита капитала ≠ risk-free; проверка fee-drag).
- **Рейтинги РФ** (ред.): нац. ≠ межд. шкала; **ВДО — рыночная конвенция, не жёсткая
  граница** (рейтинг + размер эмитента + ликвидность + спред + структура выпуска);
  предпочитать рейтинг ВЫПУСКА рейтингу эмитента; `.sf` — структурный, несопоставим
  напрямую. Кредит оценивать через **G-спред к ОФЗ совпадающей дюрации с поправками**
  на оферту/суборд/ликвидность/амортизацию/налог/устаревшую цену — не по абсолютному
  YTM и не как «чистый» кредитный спред.

### S4 — Корректность оценки и риск-метрик
Файл: `references/instruments-rf.md`, раздел «Оценка и метрики»; ссылки из доктрины.
- Формула (ред. по codex):
  `позиция_грязная = ((чистая_цена%/100) × текущий_номинал + НКД_на_бумагу) × кол-во × FX`,
  где `текущий_номинал` = непогашенный после амортизации (для ОФЗ-ИН —
  индексированный). Гварды: размер лота vs кол-во бумаг; НКД на дату расчёта;
  не применять амортизацию дважды; цена cbonds — **чистая**.
- **Рыночная оценка брокера — основной mark портфеля; формула — инструмент сверки.**
- Восстановление $-номинала: `RUB / FX / кол-во → подразумеваемый номинал`,
  привязка к легальному $100/$1000, ошибка если между → стоп.
- Sanity-проверка (нумерованный шаг): расчёт vs рублёвая оценка брокера; >10–20%
  расхождение → стоп, разобрать (номинал/FX/$100-vs-$1000/амортизация). Базис
  оценки брокера (clean/dirty, включён ли НКД) фиксировать.
- **Дюрация/доходность (ред., yield-to-worst):** считать и подписывать YTM, YTP, YTC
  и **yield-to-worst**; put как эффективное погашение — только для жёсткого
  инвесторского put с известными условиями выкупа и экономически вероятным
  исполнением. Если будущий купон после оферты неизвестен — **не схлопывать**
  длинную бумагу в короткую дюрацию без явной пометки «сценарно». Call —
  опцион эмитента (ограничивает upside, **не** защищает дюрацию инвестора): для
  премиальных смотреть YTC/YTW, для дисконтных может доминировать дюрация к погашению.
- Разделить процентную и спред-дюрацию. Флоутер: **низкая ценовая дюрация, но не
  ноль риска** (RUONIA-reset, лаг, спред, ликвидность) — смягчить «без дюрации» в
  доктрине. Реинвест-риск амортизируемых/высококупонных в цикле снижения.

- TBank: разделы по **тексту заголовка**, не по номеру; проверка полноты PDF

  разделители; колонки по **заголовку**, не offset; фолбэк xlsx-скилл.
- Обобщённый детект пустого/частичного (не только строка Сбера); покрытие дат «на
  сегодня», предупреждение о устаревании.
- Базис оценки брокера (gap по codex): clean vs dirty, включён ли НКД, дата отчёта
  vs дата котировки — зафиксировать перед использованием mark.
- Инвариант анти-дабл-каунта: из ОДНОГО авторитетного раздела на брокера; сверка

### S6 — Надёжность cbonds (браузер)
Файл: `references/cbonds-workflow.md`.
- **ISIN-gate (наивысший приоритет, по codex):** первым полем читать ISIN на
  карточке, сверять байт-в-байт с ISIN портфеля до записи любой цифры; mismatch →
  отбросить, переискать (ловушка серий эмитента — ТГК-14).
- Локаторы привязать к **реальным тулам рантайма**: `find`/`get_page_text`
  (и `javascript_tool`, если доступен) — основной способ найти поле/первый
  результат; пиксельные координаты — крайний фолбэк с пометкой «вероятно неверно
  при ресайзе». Ресайз к 1440×900, zoom=100% в начале.
- Бэнды правдоподобности (цена 1–300%, YTM/купон 0–100%, дюрация 0–30 лет);
  выход → извлечение провалено, перечитать скриншотом. Числа — скриншот по
  умолчанию, `get_page_text` для текста.
- Статус по бумаге (verified/failed/skipped); один ретрай → продолжить; в конце
  перечислить бумаги без данных cbonds.

### S7 — Налоги и структура счетов
Файл: `references/taxes-accounts.md` (новый) + шаг в SKILL.md.
- **ЛДВ (ст. 219.1, ред.):** положительный фин-результат от продажи/погашения
  обращающихся ЦБ при сроке >3 лет, лимит `3 млн ₽ × лет`; **не освобождает
  купоны**; **не действует** для бумаг на ИИС. Перед рекомендацией о продаже —
  проверка срока владения.
- **Купонный/инвест-доход (ред.):** «обычно резидентские 13% / 15% с порогом
  ~2,4 млн ₽; общий доход — пятиступенчатая шкала ст. 224; **проверяй базу/статус/
  год**». Net carry: `купон × (1 − ставка)` с этой оговоркой, не хардкод 13/15.
- **ИИС-3 (ред.):** мин. срок 5 лет для счетов 2024–2026, далее ступенью к 10
  годам к 2031; ранний вывод — не просто «форфейт», возможно восстановление
  использованных вычетов с пенями. Дать мини-таблицу «год открытия → мин. срок».
- Валютная переоценка на каждой FX/замещайке/евробонде (укрепление ₽ → налоговый
  убыток; ослабление → налогооблагаемая прибыль без $-P&L).
- **Налоговый лот/FIFO** (gap по codex): для анализа продажи и замещаек —
  учитывать порядок лотов и continuity.
- Шаг: разложить позиции по налоговой обёртке; ИИС-3 выгоден для налогонеэффективных
  имён.

### S9 — Риск-фреймворк: корзины, пороги, агрегаты (split, часть 1)
Файл: `references/analysis-doctrine.md`.
- Корзины MECE с явной precedence; 2 оси (тип риска × валюта) — FX тегается
  ортогонально, не как взаимоисключающая корзина.
- Пороги концентрации: эмитент >10%, сектор >25%, топ-5 >50%, неликвид >X%.
- Портфельные агрегаты: взвеш. дюрация, доходность (net of tax), рейтинг (по
  низшему), доля с офертой <12 мес.
- Кредит-вотч (негативный прогноз/даунгрейд) + правило секторного контагиона.
- Бакетинг по **низшему** текущему рейтингу; флаг расхождения >1 ступени.
- Инвариант: позиция в одной корзине, суммы = grand total, доли = 100% ± округл.;
  печатать строку сверки.
