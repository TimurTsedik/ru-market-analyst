## Цель
Целостный процесс «от отчётов до результата» + **обязательный выход: рекомендательная
таблица** (что убрать → на что заменить, метрики до/после, кандидаты строго из cbonds).
Точность «как в аптеке»: каждое число с источником+as-of; **текущие — точные, пост-
сделочные (after) — оценочные `*`**.

## F1. Приём отчётов и парсинг (`broker-reports.md`, SKILL.md шаг 1)
- **Структурные holdings/marks первичны (обычно Excel/XLS).** Если у брокера есть
  выбор — просить Excel, чище и часто с рыночной оценкой + сверочным итогом. **НО PDF
  не игнорировать:** из ТБанк-PDF берём сделки, кэш, купонные выплаты, удержанный
  налог, идентичность/тип счёта, признаки плеча, кросс-чек $-номинала. Тип файла — по
  содержимому, не по имени.
- **Тулчейн (порядок, venv — ФОЛБЭК):** (1) сначала доступные скиллы/системные тулзы
  (xlsx-скилл для .xlsx, pdf-скилл); (2) если нет — **локальный/временный venv**
  `pypdf` (PDF-текст) + `xlrd` (старый .xls); `pdfplumber` опционально, если важна
  структура таблиц; (3) **никогда глобальный `pip`** (PEP 668); (4) если установка/
  сеть заблокированы — попросить Excel/CSV/текстовый экспорт. PDF-текст НЕ всегда в
  контексте — извлекать явно.
- **Пустой/частичный отчёт:** детект (обобщённый) + инструкция переэкспорта (полный
  период; для Альфы — лист «Динамика позиций»).
- **Тип счёта (ИИС/брокерский)** в отчёте может отсутствовать → спрашивать (см. F6).

## F2. cbonds-цикл устойчивее (`cbonds-workflow.md`)
- **Поиск:** скроллить вверх/чистая главная → ввод ISIN → **скриншот выпадашки** →
  клик по результату; локатор `find`/DOM, пиксели — фолбэк. ISIN-gate оставить.
  get_page_text отдаёт МСФО → числа со скриншота.
- **Скачивание эмиссионных документов:** клик по ссылке может не качать → **прямой
  `navigate` на URL загрузки** (сессия залогинена) → zip → unzip → `pypdf`.
- **Детект по карточке:** «-USD»/«бивалютные» → валютная нога; «Переменная ставка»/
  «Формула расчёта купона» → флоутер; «Секьюритизация/.sf» → амортизация; «Структур.
  продукты/Дисконтные/для квалов» + цена «—» → структурка/неликвид.

## F3. Корректность инструментов (`instruments-rf.md`)
- **Амортизация секьюритизаций:** читать **и** `цена%`, **и** непогашенный номинал /
  principal factor (текущий face) — ключевая переменная именно текущий номинал, не
  «цена ≈ 63%». **Анти-двойной-счёт:** не трактовать низкую стоимость одновременно как
  низкую цену и амортизацию, если cbonds явно не показал оба. Кросс-чек с марком брокера.
- **Структурные ноты (ИОС) — порядок:** (1) cbonds-карточка + эмиссионные документы
  (идентичность + каркас); (2) если payoff неполон (нет базового актива/барьера/
  купона) → **запросить у пользователя применимые условия: скрины ноты из приложения
  (можно несколько) / term sheet / KID** — штатный источник, **только для структур/
  ИОС/ЦФА/неизвестного payoff или пробела в марке**, НЕ общий заменитель cbonds для
  обычных бумаг; (3) разобрать payoff; (4) занести в KB. **Failure-state:** если payoff
  так и неизвестен → класс «структура/неликвид, payoff unknown», НЕ считать точный
  YTW/payoff, рекомендовать только после получения условий. Цена «—» → неликвид, оценка
  по марку брокера.
- **$-номинал бивалютных — с гвардами:** восстановление через купонную выплату требует
  **gross-купон/бумага, кол-во, FX+as-of, поправку на налог, частичный период,
  амортизацию, плавающий купон**; сверять с купоном на cbonds/в проспекте. Иначе $100
  vs $1000 можно ошибиться.
- **Эквити — только live MOEX** (last/close + дата/as-of; рынок закрыт/устарело →
  пометка `*`); цена сделки = cost basis, НЕ оценка.

## F4. Рыночные данные (новый `references/market-data.md`; `sources.md` — иерархия)
- **MOEX ISS** рецепты эндпоинтов: фонды board TQTF, эквити TQBR — **но это не
  универсально**: если board неизвестен — смотреть boards-метаданные ISS; кривая —
  ликвидные ОФЗ (2/5/10-летние) + RGBI как контекст. Пример:
  `iss.moex.com/iss/engines/stock/markets/shares/securities/{SECID}.json`.
- **ЦБ:** ключевая ставка (+сигнал), курс (cbr-xml-daily) через `web_fetch`.
- Маршрут по классам: фонды/эквити/исполнимая цена → MOEX; термины/рейтинги/проспекты
  → cbonds; ставка/курс/макро → ЦБ/Минфин.

## F5. НОВЫЙ выход — рекомендательная таблица
(`analysis-doctrine.md` шаблон + `strategy-engine.md` синтез + `cbonds-workflow.md`
скринер + SKILL.md формат/DoD)

### 5a. «План замен»
| # | Убрать/сократить (бумага · ISIN · ₽ · доля) | Почему (основание cbonds) | Заменить на — **кандидат(ы) из cbonds** (можно ранжированные альтернативы; бумага · ISIN · **корзина** · рейтинг · YTW · дюрация · цена% · ликв.) | Налог.эффект* | Ликвидность/выход | Уверенность |
- Действия конкретны, но как разбор (не приказ). **Текущие суммы/доли — точные;
  целевые (после) — оценочные `*`.**
- **Сопоставимость:** показывать, в какую корзину попадает каждый кандидат (замена
  слабого кредита фондом/ОФЗ/флоутером/эквити меняет тип риска — это явно).

### 5b. «Метрики до/после»
| Метрика | До | После* | Δ |
- Взвеш. эфф./сценарная дюрация; доходность к worst **net-of-tax**; низший рейтинг и
  доля «ruBBB и ниже»; доля подушки/флоутеров; валютная нога; плечо; концентрация
  (топ-эмитент/сектор/топ-5); доля структур/неликвида.
- **Определение «После» (явно):** After = текущий портфель **минус** предложенные
  продажи/сокращения **плюс** кандидаты по **live cbonds/MOEX-маркам** (та же as-of,
  где возможно), **нетто** оценочного налога, комиссий/спреда и кэш-остатка/НКД
  (разница ACI при переключении облигаций). Все агрегаты пересчитать от пост-сделочных
  весов. Драйверы оценочности: размер лота, остаток кэша, НКД, налог, комиссия/спред.
- **Сверка стоимости (чтобы до/после не плыло):** After-стоимость портфеля =
  остаточные марки + марки замен + остаток кэша − оценочные издержки/налоги; должна
  сходиться с До (± оценочные издержки). Печатать строку сверки.

### 5c. Кандидаты — СТРОГО из cbonds (конкретный протокол)
1. Сформировать целевой профиль из интейка (рейтинг ≥ пола, дюрация-диапазон,
   senior/не суборд, ликвидность ≥ порога, без структур/дефолтов/отозванных рейтингов,
   нужная корзина/валюта).
2. **Скринер cbonds** («Поиск облигаций»/фильтры): зафиксировать фильтры + скриншот
   результата + as-of; взять 2–5 кандидатов.
3. **Открыть карточку каждого** → ISIN-gate → верифицировать: seniority/не-суборд,
   структура, статус рейтинга (нет Withdrawn), ликвидность, текущий номинал, YTW,
   дюрация, цена. Включать **только верифицированных**.
4. **Констрейнты-фильтр:** жёсткий блок — если кандидат нарушает явно заданный
   пользователем лимит (эмитент/сектор/рейтинг-пол/FX) или лок-ап ИИС. Потеря ЛДВ —
   **не жёсткий блок**: предлагать только с явной пометкой налог-эффекта и обоснованием
   (иногда снижение риска перевешивает), хард-блок лишь если пользователь задал ЛДВ
   констрейнтом.
5. Если скринер недоступен — **так и сказать**, не подставлять имена из памяти.

### 5d. Точность «как в аптеке»
Каждое material-число: источник + as-of; оценки `*`; устаревшие/смешанные даты —
пометка; суммы корзин = grand total, доли = 100%.

### 5e. Цель поиска — спросить ПЕРВОЙ (главная фишка)
До генерации спроси: **что вообще ищем** — перформанс (макс total-return при
приемлемом риске) / надёжность (качество/защита, ниже дюрация и кредит-риск) /
ультра-перформанс (агрессивно: длина/ВДО/плечо — осознанно) / баланс / доход (кэрри)
/ иное. Цель управляет генерацией кандидатов и ранжированием вариантов.

### 5f. «Что если» как ПОИСК → ТОП-5 вариантов (главная механика)
Движок не останавливается на одном плане: **собирает портфели-кандидаты** из
cbonds-проверенных бумаг под цель+констрейнты → пишет метрики каждого → стрессует
(«а что если») → **продолжает моделировать** (варьирует микс, перетест) → ранжирует
по цели → показывает **ТОП-5 вариантов**, чтобы пользователь выбрал или хотя бы понял
направление. Каждый вариант: ключевые ходы + результирующие метрики + кандидаты (ISIN
из cbonds).

### 5g. Предупреждение о токенах
Глубокий «что если» (генерация и стресс множества вариантов) — **токеноёмкий**.
Предупреди пользователя ДО запуска и спроси глубину: лёгкий in-context проход
(топ-2–3) vs полный мультиагентный `Workflow` (топ-5, состязательный стресс). Это и
есть место hybrid-эскалации движка.
