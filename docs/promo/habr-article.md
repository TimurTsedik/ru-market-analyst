## 1. Проблема

- Несколько брокеров → нет единой картины риска; концентрация по эмитенту «размазана».
- Ручной разбor облигаций (дюрация, YTW, рейтинг, оферты, амортизация) — долго и легко
  ошибиться.
- Советы «купи это» без оснований и без учёта налогов (ИИС/ЛДВ, налог на купон).

## 2. Идея решения

- Не один промпт, а **набор инвокабельных скиллов** + оркестратор: парсинг → ресёрч →
  данные → риск/стратегия.
- Главный принцип — **no-hardcode**: ставка/курс/цены/режим читаются вживую (ЦБ/MOEX/
  cbonds), не из памяти модели. (Объяснить, почему это критично для финансов.)

## 3. Архитектура (схема + список суб-скиллов)

- `broker-parse` — отчёты → позиции.
- `cbonds-research` — бумага по ISIN (карточка, рейтинг, оферта, амортизация, проспект).
- `market-data` — цена/ставка/курс/кривая (MOEX ISS, ЦБ).
- `risk-strategy` — корзины риска, детект режима, «что если» → ТОП-5 вариантов.
- `portfolio-kb` — база знаний портфеля и дифф.
- `ru-market-shared` — общий справочный пак (инструменты/источники/налоги).
- Как оркестратор связывает фазы inline-routing’ом.

## 4. Живой пример (главный «вкусный» блок)

- Вставить реальный (обезличенный) прогон: вход — пара отчётов, выход — таблица корзин,
  «План замен» и «Метрики до/после».
- Показать скриншоты/код-блоки. Подчеркнуть «кандидаты строго из cbonds, не из головы».

## 5. Интересная находка: «фонд vs прямые бумаги» (хук для обсуждения)

- БПИФ не платит налог с купона внутри фонда; прямой держатель на брокерском — платит
  ежегодно. Поэтому экономия на TER фонда часто съедается новым налогом на купон. Вывод
  **зависит от типа счёта** (брокерский vs ИИС). Это реальный кейс, где наивный расчёт
  «только TER» дал бы неверную рекомендацию.
- Хороший абзац: вовлекает аудиторию инвесторов, провоцирует комментарии.

## 6. Грабли и решения

- Парсинг разнородных отчётов (PDF/старые .xls) — фолбэк на Python (`pypdf`/`xlrd`).
- Устаревшие котировки/смешанные as-of → понижают уверенность в режиме.
- Токеноёмкость глубокого «что если» → предупреждение и выбор глубины.
- Установка соседними папками (sibling-ссылки `../...`).

## 7. Как поставить (коротко, со ссылкой на README)

- `git clone …`, разложить папки соседями в `~/.claude/skills/`, требования (cbonds,
  Claude in Chrome, web_fetch, Python).
