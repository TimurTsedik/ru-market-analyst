# Акции РФ: данные, математика, ресёрч

Канонический справочник по эквити. Питает стратегические линзы
(`../../risk-strategy/references/strategies-equity.md`) и корзины/агрегаты
(`../../risk-strategy/references/analysis-doctrine.md`). Каждое **material**-число —
source+as-of; читай вживую, не из памяти (no-hardcode). Цена сделки из отчёта — только
cost basis (`../../broker-parse/references/broker-reports.md`), не оценка.

## Источники (иерархия по типу данных)

| Что нужно | Первичный | Подтверждение / сверка |
|---|---|---|
| Цена / оборот / ликвидность / free-float / вес в индексе | MOEX ISS (рецепты — `../../market-data/references/market-data.md`) | cbonds Акции (indicative) |
| Дивиденды (объявленные / история / отсечка) | MOEX ISS `dividends` + раскрытие эмитента / IR | cbonds / smart-lab / dohod |
| Финотчётность (МСФО: P&L / BS / CF) | Эмитент IR / e-disclosure.ru | cbonds Акции (МСФО-блок) |
| Мультипликаторы (P/E, EV/EBITDA, ND/EBITDA, P/B) | **считаем сами**: market leg (цена/число акций/классы — MOEX/раскрытие) + financial leg (МСФО) | cbonds / smart-lab — сверка, не факт |
| Структура акционеров / governance | раскрытие эмитента / списки ЦБ | smart-lab |
| Buyback / выкуп | решения СД / раскрытие эмитента | новости эмитента |

- Агрегаторы (smart-lab, dohod) = **single-source label** до подтверждения; material —
  два независимых подтверждения ИЛИ явная пометка (sanity-gate — `sources.md`).
- cbonds по эквити — **indicative**; его мультипликаторы за факт не выдавай, считай сам.

## Математика (как считать)

**Мультипликаторы** — две ноги: **market leg** (live-цена, число акций по классам,
treasury/размытие — MOEX/раскрытие) + **financial leg** (прибыль/EBITDA/долг — МСФО). Из
МСФО только financial leg; весь мультипликатор считаем сами.
- `mcap = Σ(цена класса × число акций класса)`; ао/ап раздельно, treasury/размытие помечай.
- `P/E = mcap / прибыль, относимая к акционерам маткомпании` (или `price / EPS`).
- `EV = mcap + чистый долг + доли меньшинства ± прочие корректировки`. Ассоциированные/
  финвложения корректируй только согласованно с EBITDA (не входят в EBITDA → вычитай их
  equity value; учитываешь пропорц. EBITDA → пропорц. EV), не механически.
- `EV/EBITDA`, `EV/EBIT`, `ND/EBITDA = чистый долг / EBITDA`, `FCF-yield = FCF / mcap`,
  `E/P = чистая прибыль / mcap` (планка vs карри облигаций).
- **Банки:** не EV/EBITDA, а `P/B` + ROE, достаточность капитала (Н1.x), NIM, COR.

**Доходности**
- `Дивдоходность = DPS / price`.
- **Net-of-tax (обе стороны при сравнении с облигациями):** дивиденд облагается НДФЛ
  **у источника**; **ЛДВ** (льгота на прирост при продаже >3 лет) к дивидендам
  **неприменим**; на ИИС дивиденды всё равно облагаются у источника (вычет типа Б
  покрывает торговый финрез, не дивиденды). Тип счёта фиксируй; детали — `taxes-accounts.md`.
- `Buyback-yield = объём выкупа / mcap` (отметь: акции гасятся или уходят в treasury).
  `Shareholder yield = дивиденды net-of-tax + buyback-yield` — приближение: налог по
  buyback идёт через реализацию/финрез, зависит от счёта/лота, у источника не удерживается.

**Нормализация**
- База — **МСФО консолидированная**. РСБУ — только если дивполитика считается от РСБУ
  (отметь как факт по эмитенту).
- Вычищай разовое: FX-переоценка долга/статей, списания, разовые продажи активов.
- Trailing (факт) vs forward (прогноз) — помечай, не смешивай.

**Классы акций**
- ао vs ап: формула дивиденда (часто ап — фикс % прибыли по уставу), голоса (ап — без),
  ликвидность, типичный спред ап к ао. Оценивай каждый класс по своей дивбазе и
  ликвидности; держи отдельными строками.

**Total return**
- `TR = (P1 − P0 + дивиденды) / P0`, net-of-tax — для сравнения с облигационным карри.

Планка «держать акции при высокой ставке» — в `../../risk-strategy/references/strategies-equity.md`
(не дублируй; здесь — как считать входы для неё).

## Чек-лист «разобрать тикер»

1. **Identity** — secid / ISIN / борд (резолв через ISS), класс ао/ап.
2. **Цена / free-float / ликвидность** — MOEX ISS (рецепты в market-data).
3. **Дивиденды** — ISS `dividends` + раскрытие (объявленные, отсечка, история срезов).
4. **Финотчётность** — IR / e-disclosure (последняя МСФО) → посчитай мультипликаторы сам.
5. **Собственность / governance** — мажоритарий, free-float, история к миноритариям.
6. **Санкции / листинг** — статус торгов, расчёты, ограничения нерезидентов.
7. **Красные флаги** (ниже) → корзина риска + основание.

Каждый шаг: source+as-of; material → подтверждай или помечай.

## Красные флаги (эквити)

- 🚩 РСБУ-only / непрозрачное или приостановленное раскрытие.
- 🚩 Дивиденд из долга, а не из FCF (payout > FCF, рост ND/EBITDA под дивиденд).
- 🚩 Узкий free-float → гэпы, манипулируемость, риск навеса.
- 🚩 Governance: история к миноритариям, вывод прибыли через «дочки», спорные допэмиссии.
- 🚩 Серийное размытие (регулярная допэмиссия) без роста показателей на акцию.
- 🚩 Санкционные / расчётные ограничения по бумаге или эмитенту.
