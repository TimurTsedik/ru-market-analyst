# Источники данных и аудит

cbonds — основной источник по инструментам, но он сам агрегатор. Для **material**
цифр (тех, что двигают вывод или сделку) единственный источник = зацепка, не факт.
Не вешай метку источника на каждое число — только на material.

## Иерархия по типу данных

| Что нужно | Первичный источник | Подтверждение / запас |
|---|---|---|
| Стоимость портфеля (mark) | Рыночная оценка брокера (приложение/отчёт) | Расчёт по формуле (`instruments-rf.md`) — как сверка |
| Исполнимая цена, оборот, ликвидность | MOEX ISS (iss.moex.com), когда доступно | cbonds (indicative) |
| Термины: купон, оферта, амортизация, ковенанты | cbonds | Проспект (cbonds «Эмиссионные документы») |
| Рейтинги (значение) | cbonds | Сайт агентства — подтверждение действия/даты |
| Частоты дефолта / спреды по рейтингу (нац. шкала, для MC) | Эксперт РА + НКР (частоты) · АКРА (спред/рейтинг, «волна» >5%) | динамические — не хардкодить; методика — `../../risk-strategy/references/quant-scenarios.md` |
| Ставка ЦБ, RUONIA, инфляция | cbr.ru (`web_fetch`) | — |
| RGBI и индексы облигаций | MOEX (страницы индексов) | cbonds / рыночные данные |
| Размещения / план Минфина | minfin.gov.ru (`web_fetch`) | cbonds-новости |
| Дефолты, корп-события, раскрытие | e-disclosure.ru / страница раскрытия эмитента / Федресурс (где уместно) | cbonds-новости эмитента |
| Эквити (цена/дивиденды/free-float/governance; мультипликаторы — расчёт) | MOEX ISS + раскрытие/IR; **канон по эквити — `equity-fundamentals.md`** | цена сделки — cost basis; брокер-mark — сверка стоимости |

- cbonds первичен по **инструментальным** данным; **рыночная оценка брокера —
  первична по стоимости портфеля**; MOEX — по исполнимой цене/ликвидности.
- Для ставки/RGBI/Минфина заменяй сниппеты `web_search` на **`web_fetch`
  первоисточника**. Для ставки фиксируй дату вступления в силу и дату след.
  заседания ЦБ; для RGBI/Минфина — дату котировки/публикации (as-of).
- **Рецепты эндпоинтов** (MOEX ISS для фондов/эквити/кривой, ЦБ для ставки/курса) —
  в `../../market-data/references/market-data.md`.

## Без подписки cbonds (фолбэк-режим)

Подписка cbonds **желательна, но не строго обязательна**. Если её нет (или cbonds/браузер
недоступен) — работай в фолбэк-режиме, **не из памяти**:
- **Цена / YTM / дюрация / оборот / ликвидность / дивиденды / кривая ОФЗ** → **MOEX ISS**
  (рецепты — `../../market-data/references/market-data.md`): живой и бесплатный, и так первичен
  по исполнимой цене. **Free-float** — карточка бумаги MOEX / раскрытие эмитента (вес в индексе
  ISS — только прокси, не сам коэффициент).
- **Рейтинги** → **открытая часть cbonds** (что видно без подписки) + **сайты агентств**
  (Эксперт РА / АКРА / НКР публикуют рейтинги и даты действий) + приложение брокера.
- **Условия выпуска** (купон/оферта/амортизация/ковенанты) → открытая часть cbonds +
  **раскрытие эмитента** (e-disclosure.ru) + публичный проспект, если есть.
- **Кандидаты на замену** → **MOEX-скринер** (bulk по борду — market-data.md), рейтинг сверь
  по агентствам / открытому cbonds.
- **Провенанс строже:** всё из фолбэка помечай **low-confidence / single-source**; держи
  sanity-gate (два независимых подтверждения ИЛИ явная пометка). Структурки/ИОС, чей проспект
  недоступен → класс «payoff unknown», без точечного тезиса замены.

## Провенанс и свежесть

- Каждая **material** цифра несёт `источник + as-of` (дата котировки/данных из
  источника И дата чтения).
- Свежесть: `сегодня − дата котировки`. Порог-ориентир: >5 торговых дней для
  ликвидов, любая давность для неликвидов → пометь «⚠️ котировка устарела на N
  дней / последняя сделка ДД.ММ».
- **Mixed-dates:** если отчёты брокеров на разные даты — пометь итог «на смешанные
  даты ДД.ММ–ДД.ММ», не выдавай за единый срез.

## Sanity-gate перед негативным ярлыком

Прежде чем повесить «слабый кредит», «неликвид», «риск дефолта»: либо **два
независимых подтверждения** (напр. cbonds + сайт агентства), либо явная пометка
«единственный источник, не подтверждено».

Сверка рейтингов: бакетинг по **низшему** из агентств; при расхождении >1 ступени —
флаг и проверка последнего действия на сайте агентства (cbonds может отставать);
фиксируй дату и причину отзыва.

## Блок вывода «Источники и аудит»

В конце разбора — **компактный** блок по ключевым позициям/цифрам (не по каждому
числу):

```
Источники и аудит
- <Инструмент/поле>: <источник> — as-of <дата> — <confirmed | single-source | estimated>
- Отчёты брокеров: <брокер> за <период>; …
- Ставка ЦБ: X% (as-of ДД.ММ)
```

Цель — чтобы другой аналитик мог перепроверить ключевые числа.
