# RU Market Analyst

Skill для Claude, который превращает ассистента во **въедливого аналитика по рынку РФ**
(облигации, фонды, акции). На вход — брокерские отчёты (ТБанк PDF, Альфа/Сбер XLS,
скриншоты приложений). На выходе — сведённый портфель, риск-разбор по корзинам, и
**таблица рекомендаций: что убрать → на что заменить, с метриками до/после и
кандидатами строго из cbonds**.

> ⚠️ **Дисклеймер.** Скилл делает *разбор/мысль*, а не индивидуальную инвест-
> рекомендацию; Claude не лицензированный финсоветник. Размер, цена, тайминг и
> исполнение — за пользователем. Скилл только **читает** cbonds/приложения (логины не
> вводит, сделок не совершает) и **не хардкодит** рыночные данные — читает их вживую.

---

## ✨ Главное

- **🎯 Таблица рекомендаций на выходе.** Не просто «вот риски», а конкретно: что
  сократить/убрать → **на что заменить (кандидат из cbonds, не из головы)**, с
  **метриками портфеля до/после** и эффектом. Всё точно — «как в аптеке»: каждое
  число с источником и датой (as-of), оценки помечены `*`.
- **🔁 «Что если» как поиск → ТОП-5 вариантов.** Сначала спрашивает **цель**
  (перформанс / надёжность / ультра-перформанс / баланс / доход), затем собирает
  портфели-варианты из cbonds-кандидатов, гоняет стресс-сценарии, и показывает
  **топ-5 вариантов** — чтобы ты выбрал или понял направление. *(Глубокий поиск
  токеноёмкий — скилл предупреждает и спрашивает глубину.)*
- **📡 Никакого хардкода — рынок читается вживую.** Ставка/курс/цены/режим берутся из
  ЦБ/MOEX/cbonds на текущую дату, не из памяти — это снижает риск устаревания данных
  (но интерфейсы источников/правила могут меняться).
- **🔎 cbonds — первоисточник по инструментам.** Цена, рейтинги (по низшему агентству,
  с поиском отозванных), купон, оферта, амортизация, YTW/дюрация, ликвидность, новости,
  **проспекты/эмиссионные документы** (скачивает и парсит).
- **🧮 Корректная оценка.** Грязная цена + НКД + **амортизация секьюритизаций**
  (по непогашенному номиналу), восстановление **$-номинала бивалютных**, yield-to-worst
  к оферте, обязательная сверка с марком брокера (ловит ошибки «в разы»).
- **🧾 Налоги и счета.** ЛДВ, лок-ап ИИС-3, купонный налог, валютная переоценка — до
  рекомендаций о продаже.

## Как он работает (от отчётов до результата)

1. **Приём отчётов.** Если есть выбор формата — проси **Excel** (чище, часто с
   рыночной оценкой), но PDF тоже нужен (сделки, кэш, купоны, налог, тип счёта, плечо).
   Парсинг: доступные тулзы → иначе python-venv (`pypdf`/`xlrd`). Пустой отчёт —
   детект + просьба переэкспортировать.
2. **Свод по брокерам.** Агрегация одного имени/эмитента по ВСЕМ счетам и брокерам —
   до выводов о концентрации (позиция, размазанная по ТБанку и Альфе, в сумме может
   быть крупнейшей ставкой). Сверка с итогом брокера, дедуп, детект плеча.
3. **Ресёрч на cbonds (браузер).** По каждой значимой бумаге: ISIN-gate → чек-лист
   полей со скриншота. Детект типа (USD/флоутер/секьюритизация/структурка). Для
   структурных нот, где cbonds не даёт полный payoff, — запрос скринов ноты из
   приложения (штатный фолбэк).
4. **Рыночные данные.** Цены фондов/эквити и кривая — **MOEX ISS**; ставка/курс — **ЦБ**
   (`market-data.md`). Оценка портфеля первична по марку брокера.
5. **Риск-анализ.** Корзины риска (2 оси: тип риска × валюта), доли, портфельные
   агрегаты (взвеш. дюрация, доходность net-of-tax, низший рейтинг, доля валюты/плеча),
   пороги концентрации.
6. **Интейк → стратегия.** Спрашивает цель и констрейнты (тип счёта, горизонт, риск,
   лимиты), определяет **режим вживую** (форма кривой, ставка), запускает движок «что
   если» (поиск вариантов).
7. **Выход.** Таблицы **«План замен» + «Метрики до/после» + ТОП-5 вариантов**, всё с
   основаниями и дисклеймером; по запросу — рабочий файл (Excel/markdown).

### Корзины риска (2 оси: тип риска × валюта)
Заблокированные · Структура/неликвид · Слабый кредит/суборд · Длинная дюрация (фикс) ·
Real-rate (ОФЗ-ИН) · Короткий-средний фикс качества · Подушка/флоутеры · Эквити (RUB).
Поверх — валютный тег (RUB / замещайка / евробонд / юань / FX).

## Требования

| Что | Зачем | Обязательно |
|-----|-------|-------------|
| **Подписка cbonds** | Первоисточник по инструментам (цены/рейтинги/оферты/проспекты). Пользователь логинится сам. | Да, для ресёрча |
| **Claude in Chrome** | Скилл читает cbonds через залогиненный браузер пользователя. | Да, для cbonds |
| **Веб-доступ (`web_fetch`)** | MOEX ISS (цены/кривая) + ЦБ (ставка/курс). | Да |
| **Python 3 (venv)** | Парсинг PDF/старых .xls (`pypdf`/`xlrd`). | Фолбэк (если нет готовых тулзов) |
| **LibreOffice** | Альтернативная конвертация XLS (необязательна — есть python-путь). | Нет |

## Использование

Скилл **самосрабатывающий** — пиши на естественном языке: пришли отчёты, попроси
«сведи портфель / оцени риски / что поменять / какая стратегия». Он сам спросит **цель**
(перформанс/надёжность/…) и при необходимости — недостающие данные (тип счёта, скрины
структурной ноты). Перед глубоким «что если» предупредит о расходе токенов.

## Роадмап: суб-скиллы (план, ещё не реализовано)

Процесс разбит на фазы, которые планируется вынести в отдельные суб-скиллы, а главный
`ru-market-analyst` будет их оркестрировать:

- **`broker-parse`** — приём и парсинг отчётов (PDF/XLS/скрины) → чистые позиции + сверка.
- **`cbonds-research`** — браузерный цикл по ISIN (поиск, ISIN-gate, чек-лист, проспекты).
- **`portfolio-kb`** — сборка/обновление базы знаний о портфеле + дифф между прогонами.
- **`market-data`** — MOEX ISS / ЦБ (цены, кривая, ставка, курс).
- **`risk-strategy`** — корзины + режим + движок «что если» + ТОП-5.

Пока всё это работает внутри одного скилла (текущие файлы); разбиение — следующий заход.

## Принципы
- **cbonds первичен по инструментам; рыночная оценка брокера — по стоимости портфеля.**
- **Читай рынок вживую, не из памяти** (ставка/курс/цены/режим — на текущую дату).
- **Проверяй имена, не вешай ярлыки** (вердикт — после данных).
- **После каждого рассуждения — основание**; в конце — дисклеймер.
- **Кандидаты на замену — только из cbonds** (верифицированы), не из головы.
