# RU Market Analyst

Skill для Claude, который превращает ассистента во **въедливого аналитика по
российскому рынку облигаций и портфелям**. Берёт на вход брокерские отчёты
(ТБанк PDF, Альфа/Сбер XLS, скриншоты приложений), сводит позиции по всем
брокерам, проверяет каждую бумагу по **cbonds** через браузер, подтягивает
макро через веб-поиск и выдаёт риск-анализ по корзинам с конкретным планом в
рублях.

> ⚠️ **Дисклеймер.** Скилл делает *разбор*, а не индивидуальную инвестиционную
> рекомендацию. Claude не является лицензированным финансовым советником.
> Конкретные сделки и тайминг — на стороне пользователя. Авторизуется на cbonds
> и совершает сделки только сам пользователь — скилл логины/пароли не вводит и
> сделок не совершает.

---

## Что он делает

Полный цикл анализа за один проход:

1. **Парсинг отчётов** — достаёт по каждому счёту бумаги, ISIN, количества,
   цены/стоимости, кэш, валюту и **признаки плеча** (РЕПО, маржинальная торговля).
2. **Свод по брокерам** — объединяет все счета в один портфель и, что критично,
   **агрегирует одно и то же имя/ISIN по ВСЕМ брокерам** перед оценкой
   концентрации (позиция, размазанная по ТБанку и Альфе, в сумме может быть
   крупнейшей ставкой).
3. **Ресёрч на cbonds** — открывает карточку каждой значимой бумаги в браузере и
   снимает чек-лист: цена (% номинала), рейтинги всех агентств, купон, YTM,
   дюрация, амортизация, ликвидность, статус, новости эмитента, проспекты.
   Активно ищет красные флаги (отозванные рейтинги, неликвид, структурные ноты).
4. **Макро-контекст** — через веб-поиск: ключевая ставка ЦБ и траектория,
   динамика RGBI, размещения Минфина, события рынка.
5. **Риск-анализ и план** — раскладывает портфель по корзинам риска, считает доли
   от общего итога и выдаёт приоритизированный план действий в рублях.

### Корзины риска

Подушка + флоутеры · Длинная дюрация (фикс) · Короткий-средний фикс качества ·
Слабый кредит · Структура / неликвид · Валютная · Эквити (RUB).

## Требования

| Что | Зачем | Обязательно |
|-----|-------|-------------|
| **Подписка cbonds** | Первичный источник по инструментам (цены, рейтинги, новости, проспекты). Пользователь логинится сам. | Да, для полного ресёрча |
| **Claude in Chrome** (расширение для браузера) | Скилл читает cbonds через уже залогиненный браузер пользователя. | Да, для шага cbonds |
| **LibreOffice** | Конвертация бинарных XLS Альфы/Сбера в CSV (`--headless --convert-to csv`). | Только для XLS-отчётов |
| **Веб-поиск** | Макро-контекст (ставка ЦБ, RGBI, новости дня). | Желательно |

> Без cbonds и браузера скилл всё равно сведёт портфель и даст разбор по тому,
> что есть в отчётах, но без сверки рейтингов/цен по первичному источнику.

## Установка

Скилл — это папка с файлом `SKILL.md` и вложенной директорией `references/`.
Скачайте репозиторий:

```bash
git clone https://github.com/TimurTsedik/ru-market-analyst.git
```

Дальше — в зависимости от того, где используете Claude.

### Вариант 1. Claude Code (CLI) — персональный скилл

Доступен во всех проектах на этой машине.

```bash
# macOS / Linux
mkdir -p ~/.claude/skills
cp -R ru-market-analyst ~/.claude/skills/ru-market-analyst
```

Должно получиться:

```
~/.claude/skills/ru-market-analyst/
├── SKILL.md
└── references/
    ├── broker-reports.md
    ├── cbonds-workflow.md
    └── analysis-doctrine.md
```

Перезапустите сессию Claude Code (или начните новую) — скилл подхватится
автоматически.

### Вариант 2. Claude Code (CLI) — скилл уровня проекта

Если хотите, чтобы скилл жил рядом с конкретным проектом и попадал в репозиторий:

```bash
mkdir -p <ваш-проект>/.claude/skills
cp -R ru-market-analyst <ваш-проект>/.claude/skills/ru-market-analyst
```

### Вариант 3. Десктоп- или веб-приложение Claude (Claude.ai)

1. Запакуйте папку скилла в zip:
   ```bash
   cd ru-market-analyst && zip -r ../ru-market-analyst.zip . -x '.git/*'
   ```
2. В приложении Claude откройте **Settings → Capabilities → Skills**.
3. Нажмите **Upload skill** и выберите `ru-market-analyst.zip`.
4. Включите скилл.

> Для работы шага cbonds в приложении Claude должно быть подключено расширение
> **Claude in Chrome** (Connectors / Browser).

### Проверка установки

Спросите Claude: «какие скиллы тебе доступны?» — в списке должен появиться
`ru-market-analyst`. В Claude Code можно перечислить установленные скиллы и
убедиться, что папка лежит в `~/.claude/skills/`.

## Как пользоваться

Скилл **самосрабатывающий** — отдельную команду писать не нужно. Он включается,
когда вы:

- присылаете брокерский отчёт (ТБанк PDF, Альфа/Сбер XLS) или скриншот приложения;
- просите «проанализируй мои риски», «сведи портфель по брокерам», «что изменить
  в портфеле»;
- спрашиваете про конкретную облигацию / ОФЗ / флоутер / дюрацию / кредитный риск
  / доходность;
- упоминаете cbonds.

### Примеры запросов

```
Вот отчёты из ТБанка и Альфы. Сведи портфель и покажи, где у меня концентрация.
```

```
Проверь ОФЗ 26221 по cbonds — что с дюрацией и стоит ли держать?
```

```
Проанализируй мои риски по всем брокерам и дай план, что сократить.
```

### Что вы получите на выходе

Разбор в чате по фиксированной структуре:

1. Суммы по брокерам + общий итог.
2. Находки first — концентрации и красные флаги вынесены наверх.
3. Таблица корзин риска (корзина → сумма → доля → что внутри).
4. Уточнения из данных: идёт ли кэрри, валютные эффекты, плечо.
5. План по приоритету в рублях.
6. Дисклеймер.
7. Предложение собрать рабочий файл (Excel/markdown) — формат на ваш выбор.

### Подключение к cbonds (происходит в диалоге)

Когда дело доходит до ресёрча, скилл спросит, какой браузер использовать, и
попросит при необходимости разрешить домен `cbonds.ru` в расширении. Вы остаётесь
залогинены под своей подпиской — скилл только читает страницы.

## Принципы скилла

- **cbonds первичен.** Цены и рейтинги не угадываются по памяти — бумага
  открывается в браузере и факты считываются с карточки.
- **Проверяй имена, не вешай ярлыки.** Незнакомый тикер ≠ мусор; вердикт — только
  после сверки рейтинга/цены/статуса.
- **Эвристики — рабочие гипотезы, не догма.** Если данные противоречат правилу,
  скилл верит данным и говорит об этом.
- **Самокоррекция — часть качества.** Ошибся в оценке — признаёт прямо и
  пересчитывает.
