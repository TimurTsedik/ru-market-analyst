# Парсинг брокерских отчётов и свод

Цель шага — достать чистый список позиций по каждому счёту и свести в один
портфель. Брокеры дают разные форматы; ниже — что работает.

## Контроль целостности (делай всегда)

- **Сверка с итогом брокера.** Брокер печатает свой итог по счёту — бесплатная
  контрольная сумма. Сумма распарсенных позиций vs напечатанный итог: расхождение
  >1% → флаг + перечисли, что могло пропасть. Денежный остаток сверяй по оборотам/
  операциям (сделки, комиссии, налоги, переводы, купоны, конвертации, корпдействия);
  не бьётся — флаг.
- **Полнота отчёта.** Проверь, что загружен весь файл (страницы PDF, наличие
  раздела закрывающих остатков). Любой отчёт без раздела позиций или с нулём
  позиций — подозрителен (не только строка Сбера ниже). Период должен
  соответствовать задаче; если не актуален на дату анализа — пометь stale/mixed-dates
  (см. `sources.md`).
- **Базис оценки.** Если возможно, зафиксируй: рыночная оценка clean/dirty, включён
  ли НКД, дата отчёта vs дата котировки; иначе пометь basis unknown (формула —
  `instruments-rf.md`).
- **РЕПО/шорт/маржа.** Отлавливай артефакты плеча: РЕПО/РЕПО 2, отрицательные
  количества, заёмные бумаги — они могут выглядеть как позиции. Классифицируй
  отдельно (детект плеча — `analysis-doctrine.md`), не считай как обычные холдинги.
- **Анти-дабл-каунт.** Строй позиции из ОДНОГО авторитетного источника на
  брокер/счёт/дату (скриншот приложения может быть авторитетен по текущему mark,
  PDF — по остаткам на конец периода); сверь ИИС + брокерский = шапка приложения;
  дедуп по (брокер, счёт, ISIN).

## ТБанк — PDF («Отчёт о сделках и операциях»)

- Текст PDF обычно уже в контексте — читай оттуда, отдельный инструмент не нужен.
  Если PDF — скан/картинка или текст не извлёкся, используй pdf-скилл/OCR.
- **Ищи разделы по тексту заголовка, не по номеру** — нумерация плавает между
  версиями отчёта; номера ниже — лишь ориентир.
- **Раздел 1.1** — совершённые сделки (цены/количества фактических покупок-продаж;
  бери отсюда цены входа для эквити вместо рыночных оценок).
- **Раздел 2** — операции с деньгами: входящий/исходящий остаток по валютам,
  купоны/выплаты по корпдействиям (подтверждает, что кэрри идёт), налоги, выводы.
- **Раздел 3.1** — движение по ЦБ: входящий остаток → зачисление → списание →
  это и есть позиции на конец периода. Бери закрывающие количества.
- **Раздел 4.1** — справочник бумаг: ISIN, тип («ПАИ ФОНДОВ», «акции обыкн»,
  «обл прочих рез», «Стр.обл.КО рез» = структурная). Полезно для классификации.
- **Инвестор / номер счёта** — в шапке; так различаешь брокерский счёт и ИИС.
- Признаки плеча: вид сделки «РЕПО 2», контрагент-клиринг, раздел 3.5
  «удержанная комиссия» с «MARGIN \ Комиссия за маржинальную торговлю».

## Альфа — XLS

- Бинарный OLE2. Конвертируй через LibreOffice (проверь, что `libreoffice` есть и
  CSV непустой):
  `libreoffice --headless --convert-to csv:"Text - txt - csv (StarCalc)":44,34,76 file.xls`
  Проверь кириллицу (нет mojibake) и разделители (запятая-десятичная/тысячи);
  колонки находи **по заголовку**, не по фиксированному offset. Это для старого
  OLE2 `.xls`; нативный `.xlsx` парси напрямую spreadsheet-тулзой (xlsx-скилл), без
  конвертации.
- Структура: «Динамика позиций за период» → группы **Валюта / Акции / Облигации /
  Прочее** (фонды). По каждой строке: инструмент, ISIN (в следующей строке),
  кол-во и **стоимость в рублях на начало и конец периода**, место учёта (НРД/СПБ).
- Внизу — «Стоимость всех позиций, руб.» (итог по счёту) и предрасчёт НДФЛ.
- Альфа даёт уже рублёвую рыночную оценку позиций — это удобнее, чем считать
  самому; но всё равно сверь ключевые имена по cbonds (рейтинг/риск).

## Сбер / прочие XLS

- Тот же конвертер. Детект пустого/частичного — **обобщённо** (не только по строке
  Сбера): нет раздела позиций / ноль позиций / только шапка / сообщение вида «запуск
  отчёта невозможен, попробуйте позже» = **отчёт пустой**, данных нет; скажи
  пользователю и попроси перегенерировать в кабинете брокера.

## Скриншоты приложения

- Часто закрывают пробелы, которых нет в PDF/XLS: **точные рыночные стоимости
  позиций**, остатки кэша, доступное плечо («Маржиналка до …»), номинал валютных
  бумаг (через рублёвую стоимость восстанавливается $-номинал).
- Шапка приложения = общий итог по брокеру; вычитанием подсчётов суб-счетов
  (ИИС и т.п.) получаешь брокерский счёт.

## Свод

1. Список позиций по каждому счёту с рублёвой стоимостью.
2. Итог по каждому брокеру + grand total по всем брокерам.
3. **Агрегация имён по всем брокерам** — сложи одинаковые ISIN/эмитентов, прежде
   чем оценивать концентрацию (см. доктрину: кросс-брокерная концентрация).
4. Классифицируй каждую позицию по типу для корзин риска (см. analysis-doctrine).

## Оценка стоимости

- Облигации: считай по формуле из `references/instruments-rf.md` (чистая цена +
  НКД, непогашенный/индексированный номинал, ×FX). **Рыночная рублёвая оценка из
  приложения/Альфы точнее — бери её, а формулу используй как сверку.**
- Фонды (TMON, TOFZ, LQDT и т.п.): количество × цена пая (из сделок/приложения).
- Эквити: количество × **рыночная цена** (приложение/MOEX). Цена сделки из отчёта —
  только cost basis / помеченная оценка, не текущий mark; не презюмируй (ошибка с
  Т-Техно: оценил в 9 раз выше, пока не сверил с ценой сделки 350,50 ₽).
