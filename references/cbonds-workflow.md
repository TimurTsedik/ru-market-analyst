# cbonds через браузер — точная процедура

cbonds — первичный источник по инструментам. Работа идёт через Claude in Chrome
в браузере пользователя (он уже залогинен под своей подпиской). Поиск ломается на
мелочах — следуй шагам буквально.

## Подключение (один раз за сессию)

1. Подгрузи браузерные инструменты через `tool_search`: запросы вроде
   "browser navigate read page connect Chrome", "get page text", "computer
   screenshot click type", "tabs context". Это загрузит navigate,
   list_connected_browsers, select_browser, tabs_context_mcp, get_page_text,
   find, computer, browser_batch.
2. `list_connected_browsers`. Затем ОБЯЗАТЕЛЬНО спроси пользователя (через
   ask_user_input), какой браузер использовать — перечисли каждый по имени с его
   deviceId плюс опцию «открыть экран подтверждения во всех Chrome». Сам не
   выбирай. По ответу — `select_browser` с нужным deviceId.
3. `tabs_context_mcp{createIfEmpty:true}` → запомни `tabId`.
4. Доступ к домену: расширение спросит разрешение на `cbonds.ru`. Если придёт
   «Permission denied» — попроси пользователя разрешить cbonds.ru в Chrome
   (всплывашка расширения / значок → разрешить) и повтори. Сам логин не вводишь.

## Поиск бумаги по ISIN

Используй **реальные нажатия клавиш** (`computer` → `type`), а НЕ `form_input`:
автокомплит cbonds не срабатывает на программной установке значения.

Батчем (`browser_batch`) на нужном tabId:
1. `computer left_click` по полю поиска (placeholder «Быстрый поиск облигаций,
   акций, фондов, индексов, компаний, ISIN»; координаты ~[740,103], либо найди
   через `find` «search input»).
2. `computer key cmd+a` затем `key Backspace` (очистить поле).
3. `computer type` <ISIN>.
4. `computer wait 2-3` (дать прогрузиться выпадашке).
5. `computer left_click` по первому результату в выпадашке (~[555,184];
   вкладки результатов: Облигации / Акции / Индексы / Компании / ETF & Funds).
6. `computer wait 2` → `computer screenshot`.

Если клик по результату не сработал (страница не сменилась) — сделай скриншот,
найди координату результата и кликни ещё раз. Заголовок страницы и `tabId.url`
подтверждают, что открылась правильная бумага.

## Что считывать (чек-лист по бумаге)

Со сводного блока карточки сними:
- **Название, ISIN, тип** — под заголовком курсивом: «Секьюритизация, Senior
  Secured» / «Структурированные продукты, Дисконтные, Для квал. инвесторов» /
  «Бивалютные облигации» / «Переменная ставка» (= флоутер) / «Гарантированные».
- **Рейтинги** — слева: агентства (Эксперт РА, АКРА, НКР, Dagong, и т.д.) и
  значения. 🚩 **Активно ищи `Withdrawn` / «отозван»** — отзыв рейтинга это
  сигнал, особенно если отозвали несколько агентств.
- **Статус** (В обращении), **Объём**, **Размещение**, **Погашение (оферта)** —
  смотри пут/колл-оферты, они меняют дюрацию.
- **Текущий купон** (% и фикс/плавающий; у флоутера будет «Формула расчёта»).
- **Цена** — % от номинала (indicative). 🚩 Глубокий дисконт при низком купоне =
  длинная дюрация (напр. ОФЗ 26221: купон 7,7%, цена 72%, дюр 5).
- **Доходность / дюрация** — YTM (к погашению/к put) и дюрация. 🚩 YTM сильно
  выше ключевой ставки = рынок закладывает риск (кредитный или срочную премию).
- **Ликвидность** (балл). 🚩 Цена «—» = рыночной котировки нет, неликвид (типично
  для структурных нот/ИОС — выйти до погашения может не выйти).
- **НКД**, и для секьюритизаций/амортизируемых — **непогашенный номинал** (face
  может быть < 1000; критично для оценки стоимости позиции).
- Для **бивалютных/USD**: номинал в валюте ($100 или $1000 — проверь, это меняет
  оценку в разы; сверь с рублёвой стоимостью из приложения пользователя).

`get_page_text` иногда отдаёт блок финотчётности (МСФО) вместо цены — для
числовых полей надёжнее скриншот сводного блока, чем текст.

## Глубже по спорным именам

- **Динамика цены** — иконки графика на левом рейле карточки (переключение
  цена/доходность/период).
- **Новости эмитента** — со страницы эмитента / раздел новостей по бумаге.
- **Новости рынка** — верхнее меню «НОВОСТИ & RESEARCH HUB».
- **Проспект** — раздел «Эмиссионные документы» на карточке; загрузка проспектов
  доступна по подписке (она у пользователя есть). Поднимай проспект, когда нужно
  понять payoff структурной ноты, условия оферты, амортизацию, ковенанты.

## Кэш-стратегия по round-trip

Один инструмент за `browser_batch` (поиск → выбор → скриншот). Не пытайся открыть
несколько бумаг одним батчем — выпадашка по каждой своя. Координаты результата
держи ~[555–560, 184]; если съехало — скриншоть и кликай по факту.
