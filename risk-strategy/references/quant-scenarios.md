# Вероятностные сценарии и оценка (Monte Carlo, RU)

Продвинутый тир движка «а что если»: вместо точечных шоков — распределение исходов.
**Адаптировано к практике РФ** (нац. рейтинговая шкала; агентства Эксперт РА/АКРА/НКР/НРА;
G-спред к ОФЗ; замещайки через курс ЦБ). **No-hardcode:** каждый распределительный вход —
либо калиброван вживую (source+as-of), либо **флаг-допущение** + sensitivity. MC —
**вход** в ординальный скоринг (`strategy-engine.md` §5), не замена; псевдо-точности нет.

Движок-тулкит: `../quant/` (numpy/scipy, `.venv-quant`). Запуск: `python -m quant.run
portfolio.json config.json` → JSON с ES/CVaR/перцентилями/fan-chart/декомпозицией.

## Факторы и калибровка (live vs assumption)
- **Кривая (level/slope/curvature):** PCA на истории MOEX G-curve (КБД); рецепт —
  `../../market-data/references/market-data.md`. ✅ калибруется вживую.
- **Вол/возврат ставки:** Vasicek от истории ставки/кривой (CBR/MOEX). Альтернатива —
  Hull-White от живой кривой (точный старт).
- **Кредит:** см. секцию «Кредит-оверлей» (P2) — Эксперт РА/НКР частоты, АКРА спреды.
- **Флаг-допущения:** asset/recovery-корреляции, контагион, скачки, ν Student-t — sensitivity.

## Симуляция и переоценка
- Коррелированные факторные розыгрыши → терминальная кривая на горизонт → **переоценка по
  денежным потокам** каждой бумаги (не первопорядковая дюрация); дюрация/выпуклость — сверка.
- **Относительная, не абсолютная:** MC даёт **распределение относительного** total-return;
  absolute mark остаётся брокерским (`../../ru-market-shared/references/instruments-rf.md`).
  НКД / day-count / dirty-clean / **текущий (непогашенный) номинал** / индексация ОФЗ-ИН —
  на границе построения денежных потоков (face=текущий номинал, amort-график, cum_cpi с as-of),
  не внутри MC. Кредит — через G-спред к ОФЗ совпадающей дюрации (`reprice.g_spread`).
- Сид фиксирован (воспроизводимость). Число путей — из config; проверяй сходимость
  (`risk.es_standard_error`; предупреждай при малом хвостовом сэмпле).

## Выходы
- Распределение total-return → **ES/CVaR** (предпочтительно; когерентна, ES≥VaR), VaR,
  перцентили, P(loss>порог), **fan-chart**; **компонентная ES** по позиции/корзине.
- Каждый прогон: флаги модельного риска + допущений + sensitivity. Финальный скоринг —
  ординальный.

## Когда полный MC избыточен
Лёгкий мост — probability-weighted сценарии (P3): субъективные/байес-веса на малый набор
детерминированных сценариев → взвешенный ожидаемый + хвост.

## Кредит-оверлей (P2, нац. шкала)
CreditMetrics на **российской нац. шкале**: корзины по **низшему** рейтингу из Эксперт РА/
АКРА/НКР/НРА. Матрица переходов → пороги через **inverse-normal CDF** (`Φ⁻¹(PD)`); корреляция
— **однофакторная гауссова копула** на доходностях активов (не на событиях дефолта); recovery
привязан к системному фактору (отрицательная PD-recovery зависимость).
- **Сид — RU-источники:** частоты дефолта Эксперт РА (+ НКР) по корзинам нац. шкалы (1/2/3y,
  **динамические — не хардкодить**); спред-терм-структура АКРА (~308бп/категория, B→CCC 2500бп).
- **Провенанс (no-hardcode, обязателен):** каждый кредит-**риск** вход в portfolio JSON —
  пара «значение + метаданные»: `pd` + `pd_meta`, `beta` + `beta_meta`, `mean_rr` + `mean_rr_meta`,
  где `*_meta = {source, asof, agency, scale, assumption: true|false}`. Аналитик тянет частоты
  Эксперт РА/НКР **вживую** на дату анализа и проставляет as-of; recovery/корреляции/скачки —
  `assumption: true` + sensitivity (в блоке `assumptions` вывода). `run.py` требует `*_meta` для
  каждого присутствующего кредит-риск входа И полный набор `pd`/`beta`/`mean_rr` при `credit_on`
  (без тихого дефолта), иначе `ValueError`.
  `spread_bp` — детерминированный ценовой вход (G-спред к ОФЗ, `reprice.g_spread`), под этот
  guard не подпадает, но сам считается/берётся из АКРА с as-of, а не зашивается.
- **Гэп:** полная cell-by-cell матрица переходов не подтверждена → старт «дефолт/не-дефолт +
  recovery»; полная миграция — флаг-допущение. Западные матрицы — **только механизм** + флаг.
- Выход кредита идёт в ту же ES/CVaR/декомпозицию, что и ставочная нога.

## P3: RU-инструменты, хвосты, мост
- **Переоценка RU-классов (упрощённые флаг-приближения; absolute mark = брокер, MC даёт
  относительное распределение):** ОФЗ-ПК — ценовая дюрация ≈ срок до сброса
  (`floater_price_duration`, без полной модели RUONIA-пути/спреда); ОФЗ-ИН — индексация
  номинала на ИПЦ с лагом ~3 мес (`linker_cashflows`, cum_cpi с as-of, без real-yield кривой);
  замещайки/валютные — **только статический** `price_fx` (цена в валюте × курс ЦБ с as-of);
  FX как симулируемый фактор в mc **пока НЕ реализован** (флаг — учитывай как недомоделированный
  риск валютной ноги); callable/putable — эффективная/OAS-дюрация как **флаг-приближение**, не
  полный OAS-движок.
- **Жирные хвосты:** Student-t инновации (`student_t_df` > 2) + оверлей скачков (санкции/ликвидность,
  `jump_prob/jump_scale`) — прокидываются из config в `run.py`; ν и интенсивность скачков
  **флаг-допущения** (в блоке `assumptions` вывода), sensitivity. Vasicek-калибровка exact-OU,
  но симуляция пути — Euler (вспомогательный движок; первичный — PCA G-curve).
- **Мост:** `risk.prob_weighted` — веса на малый сценарный набор, когда полный MC избыточен.
