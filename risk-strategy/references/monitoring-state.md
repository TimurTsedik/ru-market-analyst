# Мониторинг, состояние, сценарии (опционально)

Эти возможности — **по запросу пользователя**, не по умолчанию. Они превращают
разовый разбор в продолжающийся.

## Файл состояния портфеля (опт-ин)

- **Только с явного согласия** пользователя сохраняй канонический файл позиций
  (ISIN-keyed): брокер/счёт, ISIN, кол-во, оценка, корзина, источник + as-of.
- В начале файла — поле **версии схемы** (для миграций). **Без протокола
  concurrency** (single-user): при повторном запуске перечитай и перезапиши целиком.
- Следующий прогон: читай файл, мёржи новые отчёты, покажи **дифф** — что появилось/
  ушло/изменилось в долях и оценке.
- Приватность: маскируй номера счетов/договоров; клади в файл только нужное для
  анализа; предупреди, где файл лежит.

## Сценарии / чувствительность

- **Ставка:** Δстоимости ≈ −модифицированная/эффективная_дюрация × Δставки ×
  стоимость, по фикс/ставочно-чувствительным ногам (**приближение**, не по всему
  портфелю вслепую). Δставки в долях: 200 бп = 0,02. Считай ±200 бп.
- **FX:** ±10–15% шок по **чистой** валютной ноге (замещайки/евробонды/юаневые —
  после рублёвых расчётов и налоговой оговорки), не по номиналу вслепую.
- **Кредит:** набросок worst-name/сектор (что будет при даунгрейде/дефолте
  крупнейшего слабого имени).

## Вотчлист и плановый ре-ресёрч

- Веди список имён под наблюдением (кредит-вотч, приближается оферта, отозван
  рейтинг). По запросу — плановый ре-pull cbonds, если доступен schedule/tooling
  (например, еженедельный health-check или календарь оферт/купонов); иначе предложи
  ручной календарь.
