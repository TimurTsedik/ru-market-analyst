# Стратегический движок (как)

Превращает диагностику в стратегию через итеративный цикл «траинга»: режим → варианты
→ «а что если» → скоринг → итерация → синтез, в связке с принципиальным анализом.
Плейбуки — в `strategies-bonds.md` и `strategies-equity.md`.

Подача: после каждого блока рассуждения — **«Основание: …»** (данные / cbonds as-of /
макро-источник / явное допущение). Один дисклеймер в конце. Скоринг **ординальный**
(++/+/0/−/−−) + грубые диапазоны со звёздочкой; без псевдо-точности.

**НИКАКОГО ХАРДКОДА под «сегодня».** Скилл обязан работать в любых рыночных условиях.
НЕ предполагай уровень ставки, режим, цены или новостной фон из памяти. Сначала
**прочитай свежие макро/новости/данные вживую** (cbr.ru/Минфин/MOEX через `web_fetch`,
новости рынка и эмитента на cbonds), определи обстановку — и только потом советуй и
гоняй лупы «а что если». Любой числовой уровень бери из живых данных с as-of.

## 0. Интейк (до генерации)

**Сначала спроси ЦЕЛЬ поиска** (главная фишка): что вообще ищем —
- **Перформанс** (макс total-return при приемлемом риске),
- **Надёжность** (качество/защита, ниже дюрация и кредит-риск),
- **Ультра-перформанс** (агрессивно: длина/ВДО/плечо — осознанно),
- **Баланс** / **Доход (кэрри)** / иное.
Цель управляет генерацией вариантов и их ранжированием.

Затем собери/уточни (если не задано — разумные дефолты с **пометкой допущение**):
горизонт, потребность в ликвидности, риск-аппетит, тип счёта (брокерский/ИИС + год
для ИИС-3), налоговые лоты/ЛДВ, лок-ап ИИС-3, желаемый RUB/FX-сплит, лимиты эмитент/
сектор, базовый сценарий/тезис. Без этого кандидаты будут красивыми, но неприменимыми.

**Предупреждение о токенах:** глубокий «что если» (генерация и стресс множества
портфелей-вариантов) — **токеноёмкий**. До запуска предупреди пользователя и спроси
глубину: лёгкий in-context проход (топ-2–3) или полный мультиагентный `Workflow`
(топ-5, состязательный стресс — см. «Эскалация»).

## 1. Детект режима

**Сначала прочитай свежие данные вживую** (не из памяти): ставка/сигнал ЦБ, инфляция,
RGBI/кривая, новостной и санкционный фон. Затем определи режим по осям ниже; для
каждой — значение + уверенность + основание. Сигналы наследуют `../../ru-market-shared/references/sources.md`: каждый
несёт as-of и флаг устаревания. **Устаревший
или единственный (single-source) material-сигнал снижает уверенность в режиме и не
может быть единственным основанием для высокоубеждённого хода** (особенно cbonds-марки
ликвидности, G-спреды, базис замещаек, потоки в акциях).

- **Цикл ставки** (рост / у пика / снижение / низко-стабильно): ключ + сигнал/
  среднесрочный прогноз ЦБ, RUONIA, инфляция/ожидания, форма кривой.
- **Форма кривой** (инверсия / плоская / нормальная / крутая): ликвидные точки ОФЗ
  или кривая MOEX (zero-coupon/constant-maturity); RGBI — поддерживающий контекст,
  не сама кривая; спред 2/10.
- **Кредитный цикл** (узкие / расширяются / широкие спреды): G-спреды по рейтинговым
  корзинам, поток размещений, дефолты/реструктуризации, рейтинговые действия.
- **Инфляция** (рост / стабильна / дезинфляция): ИПЦ, инфл.ожидания, breakeven ОФЗ-ИН.
- **FX/санкции**: USD/CNY/RUB, премии/базис замещаек, CNY-ставки, платёжные цепочки.
- **Ликвидность** (норма / стресс; вынужденные потоки): объёмы, bid-ask, ставки
  денежного рынка/РЕПО.
- **Мета «отчётность значит?»**: кросс-секционная корреляция/широта рынка,
  post-earnings abnormal return, ревизии прогнозов, дисперсия оценок/секторов,
  признаки вынужденных потоков (индексные ребалансы), доминирование санкций/госсобытий.
  → Низкая дисперсия + рулит макро/новости + вынужденные потоки = **фундаментал не
  работает** (торгуй режим/поток). Высокая дисперсия + реакция на отчёты + стабильное
  макро = **работает** (оценка/качество/отбор). Отдельно держи: качество отчётности
  (МСФО/РСБУ), разовые статьи, валютную переоценку, FCF/дивидендную способность ≠
  headline-прибыль.

Режим бывает смешанным/переходным — фиксируй неопределённость, не натягивай ярлык.

## 2. Мост от диагностики (не переделывай анализ)

Убедись, что сняты **все ключевые метрики** (форма кривой доходностей, эфф./сценарная
дюрация, YTW, G-спреды, рейтинги, корзины и агрегаты) — бери их из диагностического
слоя (`analysis-doctrine.md` / `../../ru-market-shared/references/instruments-rf.md`), не пересчитывай заново. Старт — от
текущих корзин и агрегатов. Кандидаты = движения, улучшающие что-то конкретное:
концентрацию (сверх порогов), эфф. дюрацию под режим, FX-ногу, ликвидность, налоговый
результат, ожид. доходность net-of-tax.

## 3. Генерация кандидатов (2–4)

Из плейбуков (`strategies-bonds.md` / `strategies-equity.md`) под режим и портфель.
Каждый кандидат проходит **eligibility-гейт** до стресса:
- налоговая обёртка/счёт; ЛДВ (срок >3 лет?) / FIFO-лот; лок-ап ИИС-3; валютная
  переоценка; лимиты эмитент/сектор и потребность в ликвидности из интейка.

Кандидат, нарушающий констрейнт, отбрасывается или переформулируется.

**Помимо точечных замен — собирай портфели-ВАРИАНТЫ** (целые ребалансы под цель из
§0): каждый вариант = набор ходов из cbonds-проверенных кандидатов. Варианты — единицы
сравнения для ТОП-5 (§7).

## 4. «А что если» — стресс

Прогони каждого кандидата по библиотеке сценариев и по флипу режима:
- Ставка ±200 бп; твист кривой (steepen/flatten).
- Расширение кредитных спредов **по рейтинговым корзинам** (не только дефолт слабейшего).
- Real-yield шок ОФЗ-ИН + инфляционный лаг.
- Дивидендный срез/приостановка выплат; просадка акций −20/−30% + секторный шок.
- Сырьевой шок (экспортёры); санкции/обрыв платёжной цепочки (замещайки/евробонды).
- FX ±10–15% по чистой валютной ноге; ликвидный хэйркат / невозможность выйти по mark.
- Смена режима: фундаментал вкл/выкл; санкционная эскалация/деэскалация.

Δстоимости ставочных ног считай через эфф./мод. дюрацию (приближение; Δставки в долях,
200 бп = 0,02).

## 5. Скоринг (ординальный)

**Ранжируй ВАРИАНТЫ-портфели целиком** (по пост-сделочным весам), а не только
отдельные свопы — иначе ТОП-5 выродится в список замен. Отдельные кандидаты стрессуй
где нужно, но итоговый рейтинг — на уровне портфеля под цель.

По каждому кандидату/варианту × сценарий: ++/+/0/−/−− + грубый диапазон* (Δ% ноги/портфеля).
Сведи: ожидаемый исход, **худший случай/дродаун**, net-of-tax, и **против бенчмарка**
(в той же налоговой обёртке/счёте, где возможно — иначе сравнение искажают ИИС/ЛДВ/
купонный налог/валютная переоценка): текущая аллокация + денежный рынок/депозит-прокси
+ короткий ОФЗ/флоутер + базовый сценарий пользователя. Робастное (хорошо в среднем +
не катастрофа в худшем) предпочти высоковариативному, если пользователь явно не выбрал
агрессию.

## 6. Итерация (траинг)

Возьми лидера, закрой его слабый сценарий (комбинация / частичность / балансирующая
нога), перетест. Производные/плечо/шорт — только с явного разрешения пользователя и с
анализом ликвидности/маржи/налога (по умолчанию диагностический слой считает плечо
риск-флагом).
**Стоп-рул (детерминированный):** максимум 2 итерации ИЛИ нет **материального**
улучшения. Материальное улучшение **не должно ухудшать худший случай/дродаун и не
нарушать eligibility**; засчитывается улучшение худшего случая, результата против
бенчмарка или приоритетной цели пользователя сверх порога (по умолчанию ~1% портфеля
или явный порог). Один «нотч» в одном второстепенном сценарии — лишь подтверждающий
сигнал, не достаточное основание сам по себе.

**Поиск, не один ответ:** генерируй НЕСКОЛЬКО вариантов-портфелей (разные миксы под
цель), у каждого фиксируй метрики и стресс; продолжай моделировать, пока варианты
улучшаются по цели. **Бюджет вариантов по глубине (§0):** лёгкий = ≤2–3 варианта /
1 итерация; полный = ≤5 вариантов / 2 итерации или `Workflow`. Так токен-бюджет
детерминирован.

## 7. Синтез → ТОП-5 вариантов

Выдай: режим (+уверенность) → **рекомендованный вариант** (наиболее выгодный под
ЦЕЛЬ из §0 и живой режим): «План замен» (убрать → кандидат из cbonds) + «Метрики
до/после» → сценарии (переживает/ломается) → **основание** под тезисами → дисклеймер.
Затем — **ТОП-5 вариантов**, ранжированных по цели, чтобы пользователь выбрал или
понял направление (формат таблиц — в `analysis-doctrine.md`). Глубокий принципиальный
анализ (почему, от первых принципов + данные) вплетай по ходу, не отдельным эссе.

## Эскалация в Workflow (гибрид)

По умолчанию — цикл в уме. Эскалируй в мультиагентный Workflow, когда: крупный/сложный
портфель, высокая цена решения, просьба «глубоко/тщательно» или кандидаты близки по
скорингу.

**Hard-gate:** мультиагентные линзы запускаются ТОЛЬКО после заморозки диагностики.
Передай общий пакет фактов: единый снимок портфеля, блок «Источники и аудит»,
налоговые констрейнты, набор сценариев — иначе агенты навыдумывают разные факты.
Форма: параллельная генерация (стратег-линзы: rate-first / credit-first / FX-first /
защитная) → состязательный «а что если» (скептики ломают каждого) → скоринг → синтез.

## Дисклеймер

В конце разбора: «это разбор/мысль, не приказ; основания и допущения указаны выше;
размер, цена и тайминг — за тобой».
